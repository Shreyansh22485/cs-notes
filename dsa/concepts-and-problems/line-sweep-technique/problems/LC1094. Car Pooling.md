# LC1094. Car Pooling

[LeetCode Problem Link](https://leetcode.com/problems/car-pooling/)

## Intuition & Approach
- We want to check if a car can transport all passengers without exceeding capacity at any point.
- The trips are of the form `[numPassengers, from, to]`. We use the Line Sweep (Difference Array) technique to track capacity changes:
  - Since coordinates are small ($0 \le \text{from}, \text{to} \le 1000$), declare a difference array of size 1001.
  - For each trip `[num, u, v]`, perform `diff[u] += num` and `diff[v] -= num` (passengers get off at `v`, so the load drops).
  - Scan the difference array from 0 to 1000 while maintaining a running sum. If the load exceeds `capacity` at any point, return `false`. Else, return `true`.

## Code Implementation
```cpp
class Solution {
public:
    bool carPooling(vector<vector<int>>& trips, int capacity) {
        
        map<int,int> mpp;
        for(auto& trip : trips){
            mpp[trip[1]] += trip[0];
            mpp[trip[2]] -= trip[0];
        }

        int cnt = 0;
        for(auto it = mpp.begin() ; it != mpp.end(); it++){
            cnt += it->second;
            if(cnt > capacity){
                return false;
            }
        }
        return true;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N + M)$ where $N$ is the number of trips and $M = 1000$ is the location limit.
- **Space Complexity**: $O(M)$ for the fixed-size difference array.

---
**Key Takeaways**:
- **Line Sweep for Capacity**: Entry/exit events on a timeline can be modeled as $+val$ at start and $-val$ at end coordinates, verified via a prefix sum scan.
