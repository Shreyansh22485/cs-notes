# LC731. My Calendar II

[LeetCode Problem Link](https://leetcode.com/problems/my-calendar-ii/)

## Intuition & Approach
- We want to book events without causing a triple booking (overlap of three events).
- When we book a new event `[start, end]`, we temporarily add it to a difference map `diff`: `diff[start] += 1` and `diff[end] -= 1`.
- We then traverse the map to calculate the running sum of active events. If the active events count at any point exceeds 2, it indicates a triple booking.
- If a triple booking occurs, we undo the insert (restore the previous map values) and return `false`. Otherwise, we keep the booking and return `true`.

## Code Implementation
```cpp
class MyCalendarTwo 
{
    map<int, int> mp; // key: time; val: +1 if start, -1 if end
public:
    bool book(int start, int end) 
    {
        mp[start]++;
        mp[end]--;
        int booked = 0;
        for (auto it = mp.begin(); it != mp.end(); it++) 
        {
            booked += it->second;
            if (booked == 3) 
            {
                mp[start]--;
                mp[end]++;
                return false;
            }
        }
        return true;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N \log N)$ per booking, or $O(N^2)$ overall for $N$ bookings since we traverse the map of size up to $2N$.
- **Space Complexity**: $O(N)$ to store event boundaries.

---
**Key Takeaways**:
- **Trial Booking Rollback**: Difference maps allow trial bookings by performing the insertion, validating the maximum overlap constraint, and rolling back if the validation fails.
