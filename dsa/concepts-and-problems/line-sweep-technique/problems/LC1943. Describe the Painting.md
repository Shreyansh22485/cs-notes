# LC1943. Describe the Painting

[LeetCode Problem Link](https://leetcode.com/problems/describe-the-painting/)

## Intuition & Approach
- We are given segments of the form `[start, end, color]`. We want to describe the painting by returning non-overlapping segments with their sum of colors.
- We use a difference map (`std::map`) to dynamically track boundary split points:
  - For each segment `[L, R, color]`, do `diff[L] += color` and `diff[R] -= color`.
  - Iterate through the sorted map. The keys represent all transition boundaries.
  - Maintain a running sum of colors. For each consecutive pair of keys (`prev`, `curr`), if the running sum is $> 0$, output the segment `[prev, curr, sum]`.

## Code Implementation
```cpp
class Solution {
public:
    vector<vector<long long>> splitPainting(vector<vector<int>>& segments) {
        map<int,long long> mp;
        for(auto& segment: segments){
            int start = segment[0];
            int end = segment[1];
            int color = segment[2];

            mp[start] += color;
            mp[end] -= color;
        }

        long long cumSum = 0;
        vector<vector<long long>> ans;
        for(auto it=mp.begin();next(it)!=mp.end();it++){
            cumSum += it->second;
            it -> second = cumSum;
            int left = it->first;
            int right = next(it)->first;
            long long mix = it->second;
            if(mix > 0){
                ans.push_back({left,right,mix});    
            }
        }
        return ans;

    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N \log N)$ as inserting and traversing elements in a sorted map takes logarithmic time per boundary.
- **Space Complexity**: $O(N)$ to store segment boundary coordinates in the map.

---
**Key Takeaways**:
- **Dynamic Segment Splitting**: The keys of a difference map naturally represent all active segment split points. Iterating consecutive key pairs reconstructs non-overlapping intervals.
