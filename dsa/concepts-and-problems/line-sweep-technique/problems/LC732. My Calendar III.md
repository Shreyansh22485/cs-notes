# LC732. My Calendar III

[LeetCode Problem Link](https://leetcode.com/problems/my-calendar-iii/)

## Intuition & Approach
- We want to find the maximum number of overlapping events (the $K$-booking) after adding each new event.
- We use a difference map to track event boundaries:
  - For each booking `[start, end]`, do `diff[start] += 1` and `diff[end] -= 1`.
  - Traverse the sorted map and calculate the running sum of active events. The maximum value of the running sum encountered during the traversal is the maximum $K$-booking.

## Code Implementation
```cpp
class MyCalendarThree {
public:
    map<int,int>mp;
    MyCalendarThree() {
        
    }
    
    int book(int startTime, int endTime) {
        mp[startTime]++;
        mp[endTime]--;

        int cum = 0;
        int k = 0;
        for(auto it=mp.begin();it!=mp.end();it++){
            cum += it->second;
            k = max(k,cum);
        }
        return k;
    }
};

/**
 * Your MyCalendarThree object will be instantiated and called as such:
 * MyCalendarThree* obj = new MyCalendarThree();
 * int param_1 = obj->book(startTime,endTime);
 */
```

## Complexity Analysis
- **Time Complexity**: $O(N)$ per booking, or $O(N^2)$ overall.
- **Space Complexity**: $O(N)$ to store event boundaries.

---
**Key Takeaways**:
- **Dynamic Overlap Peak**: A sorted difference map computes the global maximum overlap of intervals at any point in time.
