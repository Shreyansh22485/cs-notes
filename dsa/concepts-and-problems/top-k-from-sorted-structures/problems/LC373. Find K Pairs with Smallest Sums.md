# LC373. Find K Pairs with Smallest Sums

[LeetCode Problem Link](https://leetcode.com/problems/find-k-pairs-with-smallest-sums/)

## Intuition & Approach
- We are given two sorted arrays `nums1` and `nums2`, and want to find $K$ pairs with the smallest sums.
- This is similar to the $K$-way merge:
  - We can form $N_1$ sorted lists, where list $i$ contains pairs `(nums1[i], nums2[j])` for $j \in [0, N_2-1]$.
  - We push the first element of each list into a min-heap: `(nums1[i] + nums2[0], i, 0)`.
  - While the heap is not empty and we haven't collected $K$ pairs:
    - Pop the smallest sum pair `(sum, i, j)`.
    - Add `[nums1[i], nums2[j]]` to our results.
    - If there is a next element in `nums2` (i.e. $j + 1 < N_2$), we push the next pair `(nums1[i] + nums2[j + 1], i, j + 1)` into the min-heap.

## Code Implementation
```cpp
class Solution {
public:
    vector<vector<int>> kSmallestPairs(vector<int>& nums1, vector<int>& nums2, int k) {
        int n = nums1.size();
        int m = nums2.size();
        priority_queue<pair<int,pair<int,int>>,vector<pair<int,pair<int,int>>>,greater<pair<int,pair<int,int>>>> pq;
        set<pair<int,int>> visited;

        pq.push({nums1[0]+nums2[0],{0,0}});
        vector<vector<int>> res;
        while(res.size() != k){
            auto it = pq.top();
            pq.pop();
            int i = it.second.first;
            int j = it.second.second;
            res.push_back({nums1[i],nums2[j]});

            if(i+1 < n && !visited.contains({i+1,j})){
                visited.insert({i+1,j});
                pq.push({nums1[i+1] + nums2[j],{i+1,j}});
            }
            if(j+1 < m && !visited.contains({i,j+1})){
                visited.insert({i,j+1});
                pq.push({nums1[i] + nums2[j+1],{i,j+1}});
            }
        }
        return res;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(K \log (\min(N_1, K)))$ as the heap size is bounded by $\min(N_1, K)$.
- **Space Complexity**: $O(\min(N_1, K))$ to store entries in the priority queue.

---
**Key Takeaways**:
- **Greedy Pair Generation**: Instead of exploring all $N_1 \times N_2$ combinations, push only the active candidates into a min-priority queue to yield $K$ smallest pairs in $O(K \log K)$ time.
