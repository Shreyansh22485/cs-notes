# LC1765. Map of Highest Peak

[LeetCode Problem Link](https://leetcode.com/problems/map-of-highest-peak/)

## Intuition & Approach
- The problem asks us to assign heights to cells in a grid containing land and water such that water cells are height 0, adjacent heights differ by at most 1, and the peak height is maximized.
- To maximize heights while maintaining the adjacent cell height difference constraint of $\le 1$, the height of each cell is bound by its distance to the closest water cell.
- This is equivalent to finding the shortest path distance from any land cell to any water cell. Since there are multiple water cells, we utilize a Multi-Source BFS:
  - We initialize the BFS queue with coordinates of all water cells, set their heights to 0, and mark them as visited.
  - We process the queue layer-by-layer. For each cell popped, we check its unvisited land neighbors, assign their height as `current_height + 1`, mark them as visited, and push them to the queue.
  - This ensures every land cell is assigned its exact minimal distance to the nearest water source.

## Code Implementation
```cpp
class Solution {
public:
    typedef pair<int,int> P;
    vector<vector<int>> directions = {{-1,0},{1,0},{0,1},{0,-1}};
    vector<vector<int>> highestPeak(vector<vector<int>>& isWater) {
        int n = isWater.size();
        int m = isWater[0].size();

        queue<P>q;
        vector<vector<bool>> vis(n,vector<bool>(m,false));
        vector<vector<int>> height(n,vector<int>(m,0));


        for(int i=0;i<n;i++){
            for(int j=0;j<m;j++){
                if(isWater[i][j]){
                    vis[i][j] = true;
                    q.push({i,j});                }
            }
        }

        int level = 0;

        while(!q.empty()){
            int size = q.size();
            while(size--){
                auto [i,j] = q.front();
                q.pop();

                for(auto& dir: directions){
                    int nr = i + dir[0];
                    int nc = j + dir[1];

                    if(nr >= 0 && nr < n && nc >= 0 && nc < m && !isWater[nr][nc] && !vis[nr][nc]){
                        vis[nr][nc] = true;
                        height[nr][nc] = level + 1;
                        q.push({nr,nc});
                    }
                }
            }
            if(!q.empty())
                level ++;
        }

        return height;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N 	imes M)$, where $N$ and $M$ are grid rows and columns. Every cell is visited once and pushed/popped from the queue at most once.
- **Space Complexity**: $O(N 	imes M)$ for the height matrix, visited array, and the BFS queue.

---
**Key Takeaways**:
- **Multi-Source Queue Seeding**: Initializing BFS with all source elements concurrently computes multi-source shortest paths in linear time.
- **Manhattan Distance Grid Field**: Multi-source BFS naturally computes Manhattan distance fields on grid structures.
