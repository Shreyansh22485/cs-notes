# LC3286. Find a Safe Walk Through a Grid

[LeetCode Problem Link](https://leetcode.com/problems/find-a-safe-walk-through-a-grid/)

## Intuition & Approach
- The problem asks to find if there is a path from the top-left to the bottom-right of a grid such that the total cost (health damage) is strictly less than the starting health. Moving onto a cell $(x, y)$ incurs a health cost of `grid[x][y]` (which is either 0 or 1).
- Since the edge weights in the transition graph are exclusively 0 and 1, standard Dijkstra's algorithm would incur unnecessary overhead ($O(V \log V + E)$). Instead, we can use 0-1 BFS via a double-ended queue (`deque`) to find the shortest path in $O(V + E)$ time.
- During state relaxation:
  - If we move to a cell with cost `0`, we push the cell to the **front** of the deque (`push_front`) because it has the same level-distance.
  - If we move to a cell with cost `1`, we push it to the **back** of the deque (`push_back`) because it belongs to the next level.
  - This ensures that the queue elements remain sorted by distance, allowing us to find the minimal health damage path to the destination cell.

## Code Implementation
```cpp
class Solution {
public:
    vector<vector<int>> directions = {{-1,0},{1,0},{0,1},{0,-1}};

    bool findSafeWalk(vector<vector<int>>& grid, int health) {
        int n = grid.size();
        int m = grid[0].size();

        deque<pair<int,int>> dq;
        vector<vector<int>> dist(n, vector<int>(m, INT_MAX));

        dist[0][0] = grid[0][0];
        dq.push_front({0, 0});

        while (!dq.empty()) {
            auto [i, j] = dq.front();
            dq.pop_front();

            for (auto &dir : directions) {
                int x = i + dir[0];
                int y = j + dir[1];

                if (x >= 0 && x < n && y >= 0 && y < m) {
                    int newDist = dist[i][j] + grid[x][y];

                    if (newDist < dist[x][y]) {
                        dist[x][y] = newDist;

                        if (grid[x][y] == 0)
                            dq.push_front({x, y});
                        else
                            dq.push_back({x, y});
                    }
                }
            }
        }

        return dist[n-1][m-1] < health;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N 	imes M)$, where $N$ and $M$ are the dimensions of the grid. Each cell is processed at most once because deque elements are always sorted.
- **Space Complexity**: $O(N 	imes M)$ for the 2D distance tracking array and the deque container.

---
**Key Takeaways**:
- **0-1 BFS Pattern**: Deque-based BFS is the optimal way to find single-source shortest paths on graphs with edge weights in $\{0, 1\}$.
- **Front vs Back Insertion**: Push 0-weight moves to the front and 1-weight moves to the back to maintain queue sorting without $O(\log V)$ priority queue overhead.
