# LC1559. Detect Cycles in 2D Grid

[LeetCode Problem Link](https://leetcode.com/problems/detect-cycles-in-2d-grid/)

## Intuition & Approach
- The problem asks us to check if there is a cycle of length $\ge 4$ consisting of matching adjacent values in a 2D grid.
- We can model this grid as an undirected graph where each cell $(i, j)$ is a node, and edges connect 4-directionally adjacent cells containing the exact same character value.
- To detect cycles in an undirected graph, we can perform a Depth-First Search (DFS) traversal:
  - We track the current cell coordinates $(i, j)$ and the coordinates of the immediate parent cell $(pi, pj)$ from which we arrived.
  - While exploring matching adjacent neighbors, if we encounter a neighbor cell that has already been visited and is **not** the parent cell, a cycle is present in the grid.
  - Since cycles of matching letters must be length $\ge 4$, checking that we do not move directly back to the parent cell is sufficient to satisfy the length requirement.

## Code Implementation
```cpp
class Solution {
public:

    bool dfs(int i, int j, int pi, int pj,vector<vector<bool>>& vis,vector<vector<char>>& grid ){
        vis[i][j] = true;
        int n = grid.size();
        int m = grid[0].size();
        int dx[] = {-1,0,1,0};
        int dy[] = {0,1,0,-1};

        for(int k=0;k<4;k++){
            int r = i + dx[k];
            int c = j + dy[k];

            if(r>=0 && r< n && c>=0 && c<m && grid[r][c] == grid[i][j]){
                if(!vis[r][c]){
                    if(dfs(r,c,i,j,vis,grid) == true) return true;
                }
                else if(r != pi && c != pj ){
                    return true;
                }
            }

        }
        return false;
    }

    bool containsCycle(vector<vector<char>>& grid) {
        int n = grid.size();
        int m = grid[0].size();

        vector<vector<bool>> vis(n,vector<bool>(m,false));

        for(int i=0;i<n;i++){
            for(int j=0;j<m;j++){
                if(!vis[i][j]){
                    if(dfs(i,j,-1,-1,vis,grid) == true){
                        return true;
                    }
                }
            }

        }
        return false;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N 	imes M)$, where $N$ and $M$ are grid dimensions. We traverse each grid cell and its 4 neighbors at most once.
- **Space Complexity**: $O(N 	imes M)$ for the recursion stack and the visited state tracker matrix.

---
**Key Takeaways**:
- **Undirected Cycle Pattern**: In undirected graph cycle detection, visiting an already visited neighbor that is not the parent indicates a cycle.
- **Implicit Grid Graph Representation**: Modeling coordinate indices as vertices and value equalities as edges is a core pattern in matrix challenges.
