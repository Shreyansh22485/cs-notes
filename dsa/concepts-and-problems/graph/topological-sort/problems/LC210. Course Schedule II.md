# LC210. Course Schedule II

[LeetCode Problem Link](https://leetcode.com/problems/course-schedule-ii/)

## Intuition & Approach
- This is similar to Course Schedule I, but we need to return a valid topological ordering of courses. If a cycle exists, return an empty array.
- We use Kahn's algorithm:
  - Maintain a queue of nodes with indegree 0.
  - As we pop nodes from the queue, we append them to our output result array.
  - Decrement neighbor indegrees, pushing any that hit 0.
  - If the output array size equals the number of courses, return it. Else, return an empty array.

## Code Implementation
```cpp
class Solution {
public:
    vector<int> findOrder(int n, vector<vector<int>>& pre) {
        vector<int> adj[n];
        vector<int> indegree(n,0);
        for(auto it: pre){
            adj[it[1]].push_back(it[0]);
            indegree[it[0]]++;
        }
        queue<int> q;
        vector<int> topo;
        for(int i=0;i<n;i++){
            if(indegree[i]==0){
                q.push(i);
            }
        }
        while(!q.empty()){
            int u=q.front();
            q.pop();
            topo.push_back(u);
            for(auto v:adj[u]){
                indegree[v]--;
                if(indegree[v]==0){
                    q.push(v);
                }
            }
        }
        for(int i=0;i<n;i++){
            if(indegree[i]!=0){
                return {};
            }
        }
        return topo;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(V + E)$ where $V = N$ (courses) and $E$ is prerequisites.
- **Space Complexity**: $O(V + E)$ for adjacency list, indegrees array, and the BFS queue.

---
**Key Takeaways**:
- **Kahn's Topological Sort Ordering**: The sequence of popped nodes in Kahn's algorithm is a valid topological sort order.
