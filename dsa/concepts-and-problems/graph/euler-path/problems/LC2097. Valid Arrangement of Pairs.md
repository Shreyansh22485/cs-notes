# LC2097. Valid Arrangement of Pairs

[LeetCode Problem Link](https://leetcode.com/problems/valid-arrangement-of-pairs/)

## Intuition & Approach
- We are given pairs of numbers representing directed edges and want to arrange them such that adjacent pairs connect. This is equivalent to finding a Eulerian Path in a directed graph.
- A Eulerian Path visits every edge exactly once. It exists if:
  - At most one node has `outdegree - indegree = 1` (the start node).
  - At most one node has `indegree - outdegree = 1` (the end node).
  - All other nodes have `indegree == outdegree`.
- We calculate degrees, identify the start node (defaulting to any node with outdegree > 0), and execute Hierholzer's Algorithm:
  - Perform DFS from the start node, greedily traversing and deleting edges as we go.
  - When a node has no outgoing edges, we push it to our path stack (post-order traversal).
  - Reversing the path stack yields the correct Eulerian path sequence.

## Code Implementation
```cpp
class Solution {
public:
    void findEulerPath(int u, unordered_map<int,vector<int>>& adj, vector<int>& eulerPath){
        while(!adj[u].empty()){
            int v = adj[u].back();
            adj[u].pop_back();
            findEulerPath(v,adj,eulerPath);
        }
        eulerPath.push_back(u);
    }
    vector<vector<int>> validArrangement(vector<vector<int>>& pairs) {
        unordered_map<int,vector<int>> adj;
        unordered_map<int,int> indegree;
        unordered_map<int,int> outdegree;
        for(auto& pair: pairs){
            int u = pair[0];
            int v = pair[1];

            adj[u].push_back(v);
            indegree[v]++;
            outdegree[u]++;
        }

        int start = pairs[0][0]; // The Euler path exist but if no starting node , then it is euler circuit , therefore any node is starting node
        for(auto& it : adj){
            int node = it.first;
            int out = outdegree[node];
            int in = indegree[node];
            cout << in << " " << out << endl;
            if(out - in == 1){
                start = node;
                break;
            }
        }
        
        vector<int> eulerPath;
        findEulerPath(start,adj,eulerPath);
        reverse(eulerPath.begin(),eulerPath.end());

        vector<vector<int>> res;
        for(int i=0;i<eulerPath.size()-1;i++){
            res.push_back({eulerPath[i],eulerPath[i+1]});
        }
        return res;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(V + E)$ since each edge is visited and deleted exactly once.
- **Space Complexity**: $O(V + E)$ for adjacency list graph representation and traversal stack.

---
**Key Takeaways**:
- **Hierholzer's Algorithm**: The standard post-order traversal technique to find Eulerian Paths or Circuits in linear time by clearing edges dynamically.
