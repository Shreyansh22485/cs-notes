# LC3620. Network Recovery Pathways

[LeetCode Problem Link](https://leetcode.com/problems/network-recovery-pathways/)

## Intuition & Approach
- We want to find a path from node 0 to $N-1$ using only online nodes, such that the path cost is $\le k$ and the minimum edge weight along the path is maximized.
- This is a maximin path problem with a path length limit $k$. We can solve it using Binary Search on the Answer combined with Dijkstra's algorithm.
- We binary search the target edge score $X$ in the range $[0, \text{maxCost}]$.
- For a candidate score $mid$, we check if there is a path from $0$ to $N-1$ of length $\le k$ using only online nodes and edges with weight $\ge mid$. We verify this reachability constraint using Dijkstra's algorithm on the filtered subgraph.
- If reachable within path budget $k$, we search the right half to find a larger score ($l = mid + 1$). Otherwise, we search the left half ($r = mid - 1$).

## Code Implementation
```cpp
class Solution {
public:
    bool isPossible(int score, unordered_map<int,vector<pair<int,int>>>& adj, vector<bool>& online, long long k){
       
        priority_queue< pair<long long, int>, vector< pair<long long, int>> , greater< pair<long long, int>>> pq;
        int n = online.size();
        vector<long long> dist(n,LLONG_MAX);
        pq.push({0,0});
        dist[0] = 0;

        while(!pq.empty()){
            auto [cost, node] = pq.top();
            pq.pop();

            if(node == n-1) return true;
            if(dist[node] < cost) continue;

            for(auto& [adjNode, wt] : adj[node]){
                if(online[adjNode] && wt >= score && cost + wt <= k && cost + wt < dist[adjNode]){
                    pq.push({cost+wt , adjNode});
                    dist[adjNode] = cost+wt;
                }
            }
            
        }
        return false;
    }
    int findMaxPathScore(vector<vector<int>>& edges, vector<bool>& online, long long k) {
        unordered_map<int,vector<pair<int,int>>> adj;
        int maxCost = -1;
        for(auto& edge : edges){
            int u = edge[0];
            int v = edge[1];
            int cost = edge[2];

            maxCost = max(maxCost,cost);
            adj[u].push_back({v,cost});
        }

        int l = 0;
        int r = maxCost;
        int ans = -1;
        while(l<=r){
            int mid = r - (r-l)/2;
            
            if(isPossible(mid,adj,online,k)){
                ans = mid;
                l = mid + 1;
            }
            else{
                r = mid - 1;
            }
        }
        return ans;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(E \log E \log(\text{maxCost}))$, where Dijkstra runs in $O(E \log V)$ per binary search step.
- **Space Complexity**: $O(V + E)$ to represent the adjacency list graph representation and Dijkstra's distance arrays.

---
**Key Takeaways**:
- **Multi-Constraint Path Finding**: Combining binary search on thresholds with shortest path verification is a highly effective template for multi-constraint path optimization.
