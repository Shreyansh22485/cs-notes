# LC1091. Shortest Path in Binary Matrix

[LeetCode Problem Link](https://leetcode.com/problems/shortest-path-in-binary-matrix/)

## Intuition & Approach
- The problem asks for the shortest path from the top-left cell to the bottom-right cell in a binary matrix. We are allowed to move 8-directionally, and we can only step on cells containing `0` (clear cells).
- Since all step weights are identical (unweighted graph), Breadth-First Search (BFS) is the optimal algorithm to find the shortest path, running in linear time.
- We start our BFS from $(0,0)$. For each popped cell, we explore its 8-directional neighbors. If a neighbor is inside grid boundaries, has a value of `0`, and has not been visited, we update its distance to `current_distance + 1`, mark it as visited, and push it to our queue.
- If we reach the destination cell $(n-1, n-1)$, we return its path length. If the queue becomes empty without reaching the target, no path exists, so we return `-1`.

## Code Implementation
```cpp
class Solution {
public:
    int shortestPathBinaryMatrix(vector<vector<int>>& grid) {
        int n = grid.size();
        int m = grid[0].size();
        if(grid[0][0] == 1 || grid[n-1][m-1] == 1) return -1;
        queue<pair<int,int>>q;
        q.push({0,0});
        // vector<vector<int>> vis(n,vector<int>(m,0)); 
        // vis[0][0] = 1;

        grid[0][0] = 1; // We can use grid as vis array
        int dx[] = {-1,0,1,0,-1,1,1,-1};
        int dy[] = {0,1,0,-1,-1,1,-1,1};
        int level = 0;

        while(!q.empty()){
            int size = q.size();
            while(size--){
                auto it = q.front();
                q.pop();
                int i = it.first;
                int j = it.second;
            
                
                if(i == n-1 && j == m-1) return level+1;

                for(int k =0;k<8;k++){
                    int r = i + dx[k];
                    int c = j + dy[k];

                    if(r>=0 && r<n && c>=0 && c< m && grid[r][c] == 0){
                        grid[r][c] = 1;
                        q.push({r,c});
                    }
                }

            }
            level++;
        }

        return -1;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N \times M)$ where $N$ and $M$ are grid dimensions, as we visit each cell at most once.
- **Space Complexity**: $O(N \times M)$ for the queue container and the visited state matrix.

---
**Key Takeaways**:
- **BFS for Unweighted Graphs**: Breadth-First Search naturally guarantees finding the shortest path in unweighted graphs or grid structures.
- **8-Directional Navigation**: Using a double loop from -1 to 1 for coordinate offsets ($dx, dy$) is a clean way to explore all 8 neighbors.
