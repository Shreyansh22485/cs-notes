# LC2685. Count the Number of Complete Components

[LeetCode Problem Link](https://leetcode.com/problems/count-the-number-of-complete-components/)

## Intuition & Approach
- A connected component in an undirected graph is **complete** if there is an edge between every pair of vertices in that component.
- In a complete graph of $V$ vertices, the number of unique undirected edges is exactly:
  $$E = \frac{V \cdot (V - 1)}{2}$$
- When we traverse a component using DFS, we traverse the adjacency list of each node. Since the graph is undirected, each edge $\{u, v\}$ is visited twice (once from $u$'s adjacency list, and once from $v$'s).
- Therefore, the sum of degree sizes (total edge-visits) accumulated in DFS must be exactly $V \cdot (V - 1)$ for a complete component.
- **Algorithm**:
  - Build the adjacency list `adj`.
  - Maintain a visited array `vis`.
  - For each unvisited node, initialize `totalNodes = 0` and `totalEdges = 0`, then launch a DFS to count all nodes and edge-visits in the current connected component.
  - If `totalEdges == totalNodes * (totalNodes - 1)`, the component is complete, so we increment our counter.

## Code Implementation
```cpp
class Solution {
public:
    void dfs(int node,unordered_map<int,vector<int>>& adj,vector<bool>& vis,int& totalNodes, int& totalEdges){
        vis[node] = 1;
        totalNodes++;

        for(auto& adjNode: adj[node]){
            totalEdges++;
            if(!vis[adjNode]){
                dfs(adjNode,adj,vis,totalNodes,totalEdges);
            }
        }
    }
    int countCompleteComponents(int n, vector<vector<int>>& edges) {
        unordered_map<int,vector<int>> adj;

        for(auto edge: edges){
            adj[edge[0]].push_back(edge[1]);
            adj[edge[1]].push_back(edge[0]);

        }

        int cntc = 0;
        vector<bool> vis(n,false);
        for(int i=0;i<n;i++){
            if(!vis[i]){
                int totalNodes = 0;
                int totalEdges = 0;
                dfs(i,adj,vis,totalNodes,totalEdges);
                if(totalEdges == (totalNodes*(totalNodes-1)) ){
                    cntc++;
                }
            }
        }
        return cntc;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(V + E)$, where $V = n$ is the number of vertices and $E$ is the number of edges. We construct the adjacency list in $O(E)$ time and traverse each vertex and edge exactly once during the DFS traversal.
- **Space Complexity**: $O(V + E)$ to store the adjacency list representation, visited state tracker, and DFS recursion call stack.

---
**Key Takeaways**:
- **Double-Counting Edges in DFS**: In undirected graphs, summing degrees (total edge traversals) yields twice the number of unique edges ($2E$), making complete component validation easy to check during traversal.
- **DFS Component Characterization**: We can extract properties of individual components (node/edge counts, diameters, cycles) during a single DFS traversal pass.
