# LC886. Possible Bipartition

[LeetCode Problem Link](https://leetcode.com/problems/possible-bipartition/)

## Intuition & Approach
- The problem asks whether we can partition $N$ people into two groups such that no two people in the same group dislike each other. This is equivalent to checking if the dislike relationship graph is bipartite (2-colorable).
- We construct an adjacency list representing the dislike relationships as an undirected graph.
- A graph is bipartite if and only if it does not contain any odd-length cycles. We can determine this by coloring the graph's vertices using two colors (e.g., 0 and 1) via Depth-First Search (DFS):
  - Initialize all vertices as uncolored (using `-1`).
  - For each uncolored node, launch a DFS and color it with color `0`.
  - Traverse all adjacent nodes. If a neighbor is uncolored, recursively color it with the opposite color (`!col`).
  - If a neighbor is already colored and its color matches the current node's color, we have detected a conflict (an odd cycle), meaning bipartition is impossible.

## Code Implementation
```cpp
class Solution {
public:
    bool dfs(int node, int col,vector<vector<int>>& graph,vector<int>& color ){
        color[node] = col;
        for(auto adjNode: graph[node]){
            if(color[adjNode] == -1){
                if(dfs(adjNode,!col,graph,color)==false) return false;
            }
            else if(color[adjNode] == col){
                return false;
            }
        }
        return true;
    }
    bool isBipartite(vector<vector<int>>& graph) {
        int n = graph.size();
        vector<int> color(n,-1);
        for(int i=0;i<n;i++){
            if(color[i]==-1){
                if(dfs(i,0,graph,color) == false){
                    return false;
                }
            }
        }
        return true;
    }
    bool possibleBipartition(int n, vector<vector<int>>& dislikes) {
        vector<vector<int>> adj(n);
        for(auto vec : dislikes){
            int u = vec[0]-1;
            int v = vec[1]-1;

            adj[u].push_back(v);
            adj[v].push_back(u);
        }
        return isBipartite(adj);
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(V + E)$, where $V = N$ (number of people) and $E$ is the size of the dislikes list. We build the graph and visit each vertex and edge at most once during DFS coloring.
- **Space Complexity**: $O(V + E)$ for the adjacency list representation, color state array, and recursive call stack.

---
**Key Takeaways**:
- **Bipartite Odd Cycle Rule**: A graph is bipartite if and only if it is free of odd-length cycles.
- **Toggle State Coloring**: Coloring states with `0` and `1` can be clean-coded using `!col` or `1 - col` to represent binary partitioning.
