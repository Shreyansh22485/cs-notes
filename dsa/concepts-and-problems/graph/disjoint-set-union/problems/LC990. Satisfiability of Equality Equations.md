# LC990. Satisfiability of Equality Equations

[LeetCode Problem Link](https://leetcode.com/problems/satisfiability-of-equality-equations/)

## Intuition & Approach
- We are given equations of equality (`==`) and inequality (`!=`) relations and must determine if a valid variable assignment exists.
- Since equality is transitive, we can group equal variables into connected components.
- We perform two passes:
  - Pass 1: For each equation `x==y`, we call `union(x, y)` to merge their components.
  - Pass 2: For each equation `x!=y`, we check if they belong to the same component (`find(x) == find(y)`). If they do, we have a contradiction, so we return `false`.
  - If no contradictions are found after checking all inequalities, we return `true`.

## Code Implementation
```cpp
class Disjoint {
    private:
        vector<int> size;
        vector<int> rank;
        vector<int> parent;
    public:
        Disjoint(int n){
            size.resize(n+1);
            parent.resize(n+1);
            rank.resize(n+1,0);
            for(int i=0;i<=n;i++){
                size[i] = 1;
                parent[i] = i;
            }
        }

        int findUlParent(int node){
            if(parent[node] == node) return node;
            return parent[node]=findUlParent(parent[node]);
        }

        void unionBySize(int u, int v){
            int ulp_u =findUlParent(u);
            int ulp_v =findUlParent(v);
            if(ulp_u == ulp_v) return;

            if(size[ulp_u] < size[ulp_v]){
                parent[ulp_u] = ulp_v;
                size[ulp_v] += size[ulp_u];
            }
            else{
                parent[ulp_v] = ulp_u;
                size[ulp_u] += size[ulp_v];                
            }
        }

        void unionByRank(int u, int v){
            int ulp_u =findUlParent(u);
            int ulp_v =findUlParent(v);
            if(ulp_u == ulp_v) return;

            if(rank[ulp_u] < rank[ulp_v]){
                parent[ulp_u] = ulp_v;
            }
            else if(rank[ulp_v] < rank[ulp_u]){
                parent[ulp_v] = ulp_u;               
            }
            else{
                parent[ulp_v] = ulp_u;               
                rank[ulp_u]++;
            }
        }
};

class Solution {
public:
    bool equationsPossible(vector<string>& equations) {
        Disjoint ds(26);

        for(auto& s : equations){
            int lhs = s[0]-'a';
            int rhs = s[3]-'a';

            if(s[1] == '='){
                ds.unionBySize(lhs,rhs);
            }
        }
        for(auto& s : equations){
            int lhs = s[0]-'a';
            int rhs = s[3]-'a';

            if(s[1] == '!'){
                if(ds.findUlParent(lhs) == ds.findUlParent(rhs)) return false;
            }
        }
        return true;
        
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N + V \cdot \alpha(V))$, where $N$ is the equations size and $V = 26$ variables.
- **Space Complexity**: $O(1)$ auxiliary space as the DSU parent array is fixed to size 26.

---
**Key Takeaways**:
- **Equivalence Relations via DSU**: DSU is the standard structure for checking the consistency of transitive equivalence relations.
