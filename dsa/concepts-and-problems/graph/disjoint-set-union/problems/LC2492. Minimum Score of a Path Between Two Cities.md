# LC2492. Minimum Score of a Path Between Two Cities

[LeetCode Problem Link](https://leetcode.com/problems/minimum-score-of-a-path-between-two-cities/)

## Intuition & Approach
- The problem asks for the minimum score of a path between city 1 and city $N$. The score of a path is defined as the minimum weight of any edge (road) on that path.
- Since we can traverse any road multiple times (including going back and forth), the minimum score of a path between 1 and $N$ is simply the minimum weight of **any** edge in the connected component containing city 1 (which will also contain city $N$).
- We can find the connected component of city 1 using a Disjoint Set Union (DSU) data structure:
  - We initialize a DSU with $N$ elements.
  - We iterate through all roads `[a, b, weight]` and perform `union(a, b)` to merge their components.
  - We then iterate through all roads again. For each road `[a, b, weight]`, if both endpoints $a$ and $b$ belong to the same component as node 1 (i.e. `find(a) == find(1)`), we update our minimum score with its weight.
  - Return the minimum weight found.

## Code Implementation
```cpp
class DisjointSet{
private:
    vector<int> size;
    vector<int> parent;
public:
    DisjointSet(int n){
        size.resize(n+1,1);
        parent.resize(n+1);
        for(int i=0;i<=n;i++){
            parent[i] = i;
        }
    }
    

    int find(int node){
        if(node == parent[node]) return node;
        return parent[node] = find(parent[node]);
    }

    void unionBySize(int u, int v){
        int ulp_u = find(u);
        int ulp_v = find(v);

        if(ulp_u == ulp_v) return;

        if(size[ulp_u] < size[ulp_v]){
            parent[ulp_u] = ulp_v;
            size[ulp_v] += size[ulp_u];
        }
        else{
            parent[ulp_v] = ulp_u;
            size[ulp_u] += size[ulp_v];
        }
    }

};

class Solution {
public:
    int minScore(int n, vector<vector<int>>& roads) {
        DisjointSet ds(n);

        for(auto& road: roads){
            int a = road[0];
            int b = road[1];

            ds.unionBySize(a,b);
        }

        int score = INT_MAX;
        for(auto& road: roads){
            int a = road[0];
            int b = road[1];
            int distance = road[2];

            if(ds.find(a) == ds.find(b) && ds.find(a) == ds.find(1)){
                score = min(score,distance);
            }
        }
        return score;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(V + E \cdot \alpha(V))$, where $V = n$ (number of cities) and $E$ is the number of roads. Performing union operations on all edges takes $O(E \cdot \alpha(V))$ time, and the second validation pass takes $O(E \cdot \alpha(V))$ time.
- **Space Complexity**: $O(V)$ for the parent and size tracking arrays in the DSU.

---
**Key Takeaways**:
- **Connected Component Property**: If you can visit any node in a component multiple times, the minimum edge weight along any path is simply the minimum edge weight in the entire connected component.
- **DSU for Component Inclusion**: DSU is highly efficient at checking if an edge $(u, v)$ belongs to the component containing a specific target node (e.g. node 1) in $O(\alpha(V))$ time.
