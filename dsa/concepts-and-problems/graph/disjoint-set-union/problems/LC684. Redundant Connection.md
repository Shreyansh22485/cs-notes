# LC684. Redundant Connection

[LeetCode Problem Link](https://leetcode.com/problems/redundant-connection/)

## Intuition & Approach
- We are given a graph that started as a tree of $N$ nodes, but one extra edge was added, introducing a cycle. We need to identify that cycle-inducing edge.
- We process the edges one by one and build the graph incrementally. When we examine an edge $(u, v)$ where both endpoints are already in the same connected component, adding this edge creates a cycle. This edge is the redundant connection.
- DSU is the ideal data structure for this: for each edge, check if `find(u) == find(v)`. If yes, return the edge. Else, perform `union(u, v)`.

## Code Implementation
```cpp
class Disjoint{
    private:
        vector<int> size;
        vector<int> parent;
    
    public:
        Disjoint(int n){
            size.resize(n+1,1);
            parent.resize(n+1);
            for(int i=0;i<n+1;i++){
                parent[i]=i;
            }
        }
        int findUlParent(int node){
            if(parent[node]==node) return node;
            return parent[node] = findUlParent(parent[node]);
        }

        void unionBySize(int u, int v){
            int ulp_u = findUlParent(u);
            int ulp_v = findUlParent(v);

            if(ulp_u == ulp_v) return;
            if(size[ulp_u] < size[ulp_v]){
                parent[ulp_u] = ulp_v;
                size[ulp_v] += size[ulp_u];
            }
            else{
                parent[ulp_v] = ulp_u;
                size[ulp_u] += size[ulp_v];
            }
        }

};
class Solution {
public:
    vector<int> findRedundantConnection(vector<vector<int>>& edges) {
        int n = edges.size();
        Disjoint ds(n);
        for(auto& edge: edges){
            int u = edge[0];
            int v = edge[1];

            if(ds.findUlParent(u) == ds.findUlParent(v)) return edge;
            ds.unionBySize(u,v);
        }
        return {};
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(E \cdot \alpha(V))$ which is virtually $O(E)$ linear time.
- **Space Complexity**: $O(V)$ for DSU parent and rank array allocations.

---
**Key Takeaways**:
- **Incremental Cycle Detection**: DSU detects cycle closures in undirected graphs as edges are added dynamically, without traversing the graph.
