# LC2316. Count Unreachable Pairs of Nodes in an Undirected Graph

[LeetCode Problem Link](https://leetcode.com/problems/count-unreachable-pairs-of-nodes-in-an-undirected-graph/)

## Intuition & Approach
- The problem asks for the total pairs of nodes that cannot reach each other. If we know the sizes of all connected components, say $S_1, S_2, \dots, S_c$, then any node in component $i$ is unreachable from $N - S_i$ nodes.
- Summing this over all components and dividing by 2 (for unordered pairs) gives the result. Alternatively, we can accumulate components: as we iterate through component sizes, we multiply the size of the current component by the sum of sizes of all previously visited components.
- We can compute connected component sizes using DSU by merging all edges, and then counting children under each representative parent.

## Code Implementation
```cpp
class Disjoint{
    private:
    vector<int> parent;
    vector<int> size;

    public:

    Disjoint(int n){
        size.resize(n+1,1);
        parent.resize(n+1);
        for(int i=0;i<n+1;i++){
            parent[i]=i;
        }
    }

    int findUlParent(int node){
        if(parent[node]==node) return node;
        return parent[node]=findUlParent(parent[node]);
    }

    void unionBySize(int u,int v){
        int ulp_u=findUlParent(u);
        int ulp_v=findUlParent(v);
        if(ulp_u==ulp_v) return;
        if(size[ulp_u]<size[ulp_v]){
            parent[ulp_u]=ulp_v;
            size[ulp_v]+=size[ulp_u];
        }
        else{
            parent[ulp_v]=ulp_u;
            size[ulp_u]+=size[ulp_v];
        }

    }
    int getSize(int node){
        return size[node];
    }

};
class Solution {
public:
    long long countPairs(int n, vector<vector<int>>& edges) {
        if(edges.size() == 0) return 1LL*n*(n-1)/2;
        Disjoint ds(n);
        for(auto& edge : edges){
            int u = edge[0];
            int v = edge[1];
            ds.unionBySize(u,v);
        }
    //      unordered_map<int,int> st;
    //     for(int i=0;i<n;i++){
    //         st[ds.findUlParent(i)] = ds.getSize(ds.findUlParent(i));
    //     }
    //     if(st.size() == 1) return 0;
    //     long long ans = 0;
    //    for(auto i = st.begin(); i != st.end(); ++i){
    //         auto j = i;
    //         ++j;
    //         for(; j != st.end(); ++j){
    //             ans += 1LL * i->second * j->second;
    //         }
    //     }

    // Better
        unordered_map<int,int> mpp;
        for(int i=0;i<n;i++){
            mpp[ds.findUlParent(i)]++;
        }
        long long ans = 0;
        long long rem = n;
        for(auto& [ulp,size] : mpp){
            ans += 1LL*size * (rem-size);
            rem -= size;
        }
        return ans;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(V + E \cdot \alpha(V))$ to group nodes and calculate component sizes.
- **Space Complexity**: $O(V)$ for DSU parent, size, and component size array storage.

---
**Key Takeaways**:
- **Unreachable Pairs Counting**: The number of unreachable pairs in a graph with partitioned component sizes is $\sum_{i < j} S_i \times S_j$, which can be computed in linear time $O(C)$ using a running prefix sum.
