# LC1319. Number of Operations to Make Network Connected

[LeetCode Problem Link](https://leetcode.com/problems/number-of-operations-to-make-network-connected/)

## Intuition & Approach
- The problem asks for the minimum cable adjustments to connect all computers in a network. To connect $C$ separate components, we need exactly $C - 1$ extra cables.
- We can only move cables if we have redundant cables. A tree of size $N$ requires exactly $N - 1$ edges, so any additional edges are redundant.
- Thus, we first check if `connections.size() < n - 1`. If so, we do not have enough cables, so we return `-1`.
- Otherwise, we use a Disjoint Set Union (DSU) structure to group computers into connected components. We initialize DSU of size $N$, and for each connection $(u, v)$, we perform `union(u, v)`.
- After processing all connections, we count the number of components $C$ (where `parent[i] == i`). The answer is $C - 1$.

## Code Implementation
```cpp
class Disjoint{
    private:
    vector<int> parent;
    vector<int> size;
    vector<int> rank;

    public:
    Disjoint(int n){
        rank.resize(n+1,0);
        size.resize(n+1,1);
        parent.resize(n+1);
        for(int i=0;i<n+1;i++){
            parent[i]=i;
        }
    }
    int findUlParent(int node){
        if(parent[node]==node) return node;
        return parent[node]=findUlParent(parent[node]);
    }
    void unionByRank(int u,int v){
        int ulp_u=findUlParent(u);
        int ulp_v=findUlParent(v);
        if(ulp_u==ulp_v) return;
        if(rank[ulp_u] < rank[ulp_v]){
            parent[ulp_u]=ulp_v;
        }
        else if(rank[ulp_v] < rank[ulp_u]){
            parent[ulp_v]=ulp_u;
        }
        else{
            parent[ulp_v]=ulp_u;
            rank[ulp_u]++;
        }

    }
    void unionBySize(int u, int v){
        int ulp_u=findUlParent(u);
        int ulp_v=findUlParent(v);
        if(ulp_u==ulp_v) return;
        if(size[ulp_u] < size[ulp_v]){
            parent[ulp_u]=ulp_v;
            size[ulp_v]+=size[ulp_u];
        }
        else{
            parent[ulp_v]=ulp_u;
            size[ulp_u]+=size[ulp_v];
        }
    }

};
class Solution {
public:
    int makeConnected(int n, vector<vector<int>>& connections) {
        int cntExtra=0;
        Disjoint ds(n);
        for(auto it:connections){
            int u=it[0];
            int v=it[1];
            if(ds.findUlParent(u)==ds.findUlParent(v)){
                cntExtra++;
            }
            else{
                ds.unionBySize(u,v);
            }
        }
        int cntC=0;
        for(int i=0;i<n;i++){
            if(ds.findUlParent(i)==i){
                cntC++;
            }
        }
        if(cntExtra >= cntC-1) return cntC-1;
        return -1;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N + E \cdot \alpha(N))$, where $E$ is connections size and $\alpha$ is the inverse Ackermann function (practically linear).
- **Space Complexity**: $O(N)$ to store the DSU parent and rank tracking arrays.

---
**Key Takeaways**:
- **Connectivity and Trees**: Connecting $C$ components always requires $C-1$ edges. DSU is the standard structure for dynamic connectivity and component tracking.
