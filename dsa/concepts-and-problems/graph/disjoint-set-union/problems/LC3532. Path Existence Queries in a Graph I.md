# LC3532. Path Existence Queries in a Graph I

[LeetCode Problem Link](https://leetcode.com/problems/path-existence-queries-in-a-graph-i/)

## Intuition & Approach
- The problem asks us to determine if there exists a path between two nodes $u$ and $v$ in a graph under specific edge constraints.
- An edge exists between node $i$ and some predecessor node if the difference in their values from `nums` does not exceed `maxDiff`.
- We can determine connectivity using a Disjoint Set Union (DSU) data structure:
  - For each index $i$ from 1 to $N-1$, we want to find if there is a valid transition node.
  - We binary search the first index `idx` in `nums` where the value is at least `nums[i] - maxDiff` using `lower_bound`.
  - If `idx < i`, we merge node $i$ with node `idx` using `unionBySize(i, idx)`.
- For each query `[u, v]`, we check if both nodes share the same representative parent in the DSU (`find(u) == find(v)`). If they do, a path exists, so we record `true`; otherwise, we record `false`.

## Code Implementation
```cpp
class DisjointSet{
private:
    vector<int> size;
    vector<int> parent;
public:
    DisjointSet(int n){
        size.resize(n+1,1);
        parent.resize(n+1);
        for(int i=0;i<=n;i++){
            parent[i] = i;
        }
    }

    int find(int node){
        if(node == parent[node]) return node;
        return parent[node] = find(parent[node]);
    }

    void unionBySize(int u, int v){
        int ulp_u = find(u);
        int ulp_v = find(v);
        if(ulp_u == ulp_v) return;

        if(size[ulp_u] < size[ulp_v]){
            parent[ulp_u] = ulp_v;
            size[ulp_v] += size[ulp_u];
        }
        else{
            parent[ulp_v] = ulp_u;
            size[ulp_u] += size[ulp_v];            
        }
    }
};

class Solution {
public:
    vector<bool> pathExistenceQueries(int n, vector<int>& nums, int maxDiff, vector<vector<int>>& queries) {
        DisjointSet ds(n);
        for(int i=1;i<n;i++){
            int idx = lower_bound(nums.begin(),nums.end(),(nums[i]-maxDiff)) - nums.begin();
            if(idx < i) ds.unionBySize(i,idx);
        }
        vector<bool> ans;
        for(auto& query: queries){
            int u = query[0];
            int v = query[1];

            if(ds.find(u) == ds.find(v)) ans.push_back(true);
            else ans.push_back(false);
        }
        return ans;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N \log N + Q \cdot \alpha(N))$, where $N$ is the number of elements in `nums` and $Q$ is the number of queries. For each of the $N$ nodes, we run a binary search `lower_bound` taking $O(\log N)$ time, and for each of the $Q$ queries, DSU lookups run in near-constant $O(\alpha(N))$ time.
- **Space Complexity**: $O(N)$ auxiliary space for the parent and size tracking arrays in the DSU.

---
**Key Takeaways**:
- **Implicit Graph Reduction**: When edges are defined by value differences on ordered datasets, combining binary search (`lower_bound`) with incremental DSU merges eliminates the need for $O(N^2)$ brute-force edge validation.
- **Connectivity Queries via DSU**: Answering reachability queries is highly efficient using DSU `find` operations, completing in $O(\alpha(N))$ time.
