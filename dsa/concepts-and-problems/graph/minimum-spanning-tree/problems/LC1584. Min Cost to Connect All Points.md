# LC1584. Min Cost to Connect All Points

[LeetCode Problem Link](https://leetcode.com/problems/min-cost-to-connect-all-points/)

## Intuition & Approach
- We want to connect all points in a 2D plane with minimum total Manhattan distance. This is a Minimum Spanning Tree (MST) problem on a complete graph (dense graph).
- We use Prim's algorithm which is highly suited for dense graphs:
  - Start at point 0. Maintain a min-heap storing `(distance, node)` and a visited set.
  - In each step, extract the minimum edge from the heap. If the node is already visited, skip it.
  - Otherwise, add its weight to the total MST cost and mark it as visited.
  - Explore all other nodes: for any unvisited node $v$, push `(ManhattanDistance(u, v), v)` to the min-heap.

## Code Implementation
```cpp
class Solution {
public:
    typedef pair<int,int> P;
    int minCostConnectPoints(vector<vector<int>>& points) {
        int n = points.size();
        map<P,bool> inMST;
        priority_queue<pair<int,P>, vector<pair<int,P>>, greater<pair<int,P>>> pq;

        P source = {points[0][0],points[0][1]};
        pq.push({0,source});

        int sum = 0;

        while(!pq.empty()){
            auto [wt, pointInfo] = pq.top();
            pq.pop();

            int x = pointInfo.first;
            int y = pointInfo.second;

            if(inMST[{x,y}] == true) continue;

            sum += wt;
            inMST[{x,y}] = true;

            for(auto& point: points){
                int x1 = point[0];
                int y1 = point[1];

                if(x1 == x && y1 == y) continue;
                if(!inMST[{x1,y1}]){
                    int val = abs(x1-x) + abs(y1-y);
                    pq.push({val,{x1,y1}});
                }
            }
        }

        return sum;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N^2 \log N)$ where $N$ is the number of points, using a min-heap priority queue on the dense graph.
- **Space Complexity**: $O(N^2)$ in the worst case to store edges in the priority queue.

---
**Key Takeaways**:
- **Prim's MST greedy expansion**: Grown node-by-node from a starting source, Prim's is optimal for dense graphs.
