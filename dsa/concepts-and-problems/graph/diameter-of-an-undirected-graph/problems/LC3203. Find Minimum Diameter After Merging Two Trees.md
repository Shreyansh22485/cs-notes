# LC3203. Find Minimum Diameter After Merging Two Trees

[LeetCode Problem Link](https://leetcode.com/problems/find-minimum-diameter-after-merging-two-trees/)

## Intuition & Approach
- We have two trees and want to connect them by drawing a single edge between a node in Tree 1 and a node in Tree 2 such that the diameter of the merged tree is minimized.
- Let $D_1$ be the diameter of Tree 1 and $D_2$ be the diameter of Tree 2.
- When connecting Tree 1 and Tree 2, the new diameter will be the maximum of:
  1. The path lies entirely within Tree 1 ($D_1$).
  2. The path lies entirely within Tree 2 ($D_2$).
  3. The path crosses the new edge. To minimize this cross-tree path, we must connect the centroids of both trees. The maximum distance from the centroid to any leaf in Tree $k$ is $\lceil D_k / 2 \rceil = \lfloor (D_k + 1)/2 \rfloor$. The merged path length is $\lceil D_1 / 2 \rceil + 1 + \lceil D_2 / 2 \rceil$.
- Thus, the merged tree's minimal diameter is $\max(D_1, D_2, \lceil D_1 / 2 \rceil + 1 + \lceil D_2 / 2 \rceil)$.
- To find a tree's diameter in linear time, we use the standard double-BFS method:
  1. Perform BFS from an arbitrary starting node (e.g., `0`) to find the farthest node $X$. Node $X$ represents one end of the diameter path.
  2. Perform BFS from node $X$ to find the farthest node $Y$. The distance to $Y$ is the tree's diameter.

## Code Implementation
```cpp
class Solution {
public:
    int findDiameter(vector<int> adj[],int n){
        auto [farthestNode,_] = findFarthestNode(adj,0,n);
        auto [_,diameter] = findFarthestNode(adj,farthestNode,n);

        return diameter;
    }
    pair<int,int> findFarthestNode(vector<int> adj[], int sourceNode, int n){
        queue<int> q;
        vector<bool> visited(n,false);
        int distance = 0;
        int farthestNode = sourceNode;

        q.push(sourceNode);
        visited[sourceNode] = true;

        while(!q.empty()){
            int size = q.size();
            while(size--){
                int node = q.front();
                q.pop();
                farthestNode = node;
                for(auto adjNode : adj[node]){
                    if(!visited[adjNode]){
                        visited[adjNode] = true;
                        q.push(adjNode);
                    }
                }
            }

            if(!q.empty()){
                distance++;
            }
        }
        return {farthestNode,distance};

    }
    int minimumDiameterAfterMerge(vector<vector<int>>& edges1, vector<vector<int>>& edges2) {
        int n1 = edges1.size()+1;
        int n2 = edges2.size()+1;

        vector<int> adj1[n1];
        for(auto edge:edges1){
            int u = edge[0];
            int v = edge[1];

            adj1[u].push_back(v);
            adj1[v].push_back(u);
        }
        vector<int> adj2[n2];
        for(auto edge:edges2){
            int u = edge[0];
            int v = edge[1];

            adj2[u].push_back(v);
            adj2[v].push_back(u);
        }
        int diameter1 = findDiameter(adj1,n1);
        int diameter2 = findDiameter(adj2,n2);

        return max({diameter1,diameter2,(diameter1+1)/2 + 1 + (diameter2+1)/2});
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(V_1 + E_1 + V_2 + E_2)$, where $V_k$ and $E_k$ represent the nodes and edges of the two trees. We execute standard linear BFS traversals on both trees.
- **Space Complexity**: $O(V_1 + E_1 + V_2 + E_2)$ to store the adjacency lists and the BFS tracking queues.

---
**Key Takeaways**:
- **Double BFS Tree Diameter**: An arbitrary node's farthest node is a diameter endpoint; searching from that endpoint yields the full diameter.
- **Optimal Centroid Connection**: The optimal joining node for any tree under diameter constraints is its midpoint/centroid.
