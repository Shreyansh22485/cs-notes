# LC1589. Maximum Sum Obtained of Any Permutation

[LeetCode Problem Link](https://leetcode.com/problems/maximum-sum-obtained-of-any-permutation/)

## Intuition & Approach
- We want to find a permutation of `nums` that maximizes the sum of elements requested across multiple queries of the form `[start, end]`.
- By greedy observation, we should place the largest numbers in `nums` at the indices that are requested most frequently.
- We use a difference array to compute the query frequency for each index in $O(N + Q)$ time:
  - For each query `[L, R]`, do `diff[L]++` and `diff[R+1]--`.
  - Compute the prefix sum of `diff` to get the frequency array `freq`.
- Sort both `freq` and `nums` in ascending order. Multiply matching indices and sum them up modulo $10^9 + 7$.

## Code Implementation
```cpp
class Solution {
public:
    const int MOD = 1e9 + 7;
    int maxSumRangeQuery(vector<int>& nums, vector<vector<int>>& requests) {
        int n = nums.size();
        vector<int> diff(n);

        for(auto& req: requests){
            int start = req[0];
            int end = req[1];

            diff[start] += 1;
            if(end+1 < n)
                diff[end+1] -= 1;
        }

        for(int i=1;i<n;i++){
            diff[i] += diff[i-1];
        }

        sort(diff.begin(),diff.end(),greater<>());
        sort(nums.begin(),nums.end(),greater<>());

        int totalSum = 0;

        for(int i=0;i<n;i++){
            totalSum = (totalSum + (diff[i]*1ll*nums[i])) % MOD;
        }
        return totalSum;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N \log N + Q)$ since sorting the frequency and numbers arrays takes $O(N \log N)$ time, and query processing takes $O(Q)$ time.
- **Space Complexity**: $O(N)$ to store the frequency tracking array.

---
**Key Takeaways**:
- **Greedy Frequency Allocation**: Use a difference array to calculate frequency weights of query overlap, and pair largest values with highest frequencies.
