# LC2848. Points That Intersect With Cars

[LeetCode Problem Link](https://leetcode.com/problems/points-that-intersect-with-cars/)

## Intuition & Approach
- We are given car coordinates `[start, end]` representing intervals. We want to count the number of integer points covered by at least one car.
- Since coordinates are small ($L, R \le 100$), we can use a difference array of size 102:
  - For each car `[L, R]`, do `diff[L]++` and `diff[R + 1]--`.
  - Compute the prefix sum of `diff`.
  - Loop from 1 to 100: if the prefix sum at index $i$ is $> 0$, increment the points counter.

## Code Implementation
```cpp
class Solution {
public:
    int numberOfPoints(vector<vector<int>>& nums) {
        vector<int> mp(102,0);
        for(auto& num : nums){
            mp[num[0]]++;
            mp[num[1]+1]--;
        }
        int ans = 0;
        int sum = 0;
        for(int i =1;i<=100;i++){
            sum+=mp[i];
            if(sum>0) ans++;
        }
        return ans;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N + M)$ where $N$ is the number of cars and $M = 100$ is the coordinate limit.
- **Space Complexity**: $O(M)$ for the fixed-size difference array.

---
**Key Takeaways**:
- **Interval Coverage**: Difference arrays are perfect for counting intervals of active range intersections on coordinate systems.
