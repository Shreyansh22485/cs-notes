# LC3346. Maximum Frequency of an Element After Performing Operations I

[LeetCode Problem Link](https://leetcode.com/problems/maximum-frequency-of-an-element-after-performing-operations-i/)

## Intuition & Approach
- We want to maximize the frequency of any element in an array after changing elements. An element $nums[i]$ can be converted to any value in $[nums[i] - k, nums[i] + k]$, with at most `numOperations` changes.
- If we target a final value $X$, $nums[i]$ can be converted to $X$ if $X \in [nums[i] - k, nums[i] + k]$.
- Each element $nums[i]$ defines a valid target range $[nums[i] - k, nums[i] + k]$. We use a difference map (using a sorted map `std::map` to handle large values without coordinate compression) to compute how many elements can reach each coordinate $X$:
  - For each element, insert interval bounds: `diff[num - k]++` and `diff[num + k + 1]--`.
  - Traverse the map to compute running prefix sums, tracking the active overlapping intervals.
  - The maximum frequency at $X$ is `original_freq[X] + min(numOperations, active_intervals - original_freq[X])`.

## Code Implementation
```cpp
class Solution {
public:
    int maxFrequency(vector<int>& nums, int k, int numOperations) {
        int maxVal = *max_element(begin(nums), end(nums)) + k;

        vector<int> diff(maxVal+2, 0);
        unordered_map<int, int> freq;

        for(int i = 0; i < nums.size(); i++) {
            freq[nums[i]]++;

            int l = max(nums[i]-k, 0);
            int r = nums[i]+k;

            diff[l]++;
            diff[r+1]--;
        }


        int result = 1;

        for(int target = 0; target <= maxVal; target++) {
            diff[target] += (target > 0 ? diff[target-1] : 0);

            int targetFreq     = freq[target];
            int needConversion = diff[target] - targetFreq;

            int maxPossibleFreq = min(needConversion, numOperations);

            result = max(result, targetFreq + maxPossibleFreq);
        }

        return result;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N \log N)$ as inserting and traversing elements in a sorted map takes logarithmic time per element.
- **Space Complexity**: $O(N)$ for the frequency map and difference map entries.

---
**Key Takeaways**:
- **Sorted Map for Large Coordinates**: When coordinates are large or sparse, using a sorted map to simulate difference updates `map[val]++` and `map[val+1]--` yields coordinate-independent prefix scanning.
