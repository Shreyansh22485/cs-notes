# LC2536. Increment Submatrices by One

[LeetCode Problem Link](https://leetcode.com/problems/increment-submatrices-by-one/)

## Intuition & Approach
- We are given an $N \times N$ grid and queries to increment submatrices of the form `[row1, col1, row2, col2]` by 1.
- We can model this using row-by-row 1D difference arrays:
  - For each query, we loop through the rows from `row1` to `row2`.
  - For a row $r$, apply 1D difference updates: `diff[r][col1] += 1` and `diff[r][col2 + 1] -= 1`.
  - After processing all queries, compute the prefix sums row-wise to reconstruct the final grid.

## Code Implementation
```cpp
class Solution {
public:
    vector<vector<int>> rangeAddQueries(int n, vector<vector<int>>& queries) {
        vector<vector<int>> diff(n, vector<int>(n, 0));

        //Same as Difference Array Technique in 1D Array

        //Step-1 Process query
        for(auto &query : queries) {
            int row1 = query[0];
            int col1 = query[1];

            int row2 = query[2];
            int col2 = query[3];

            for(int i = row1; i <= row2; i++) {
                diff[i][col1] += 1;

                if(col2 + 1 < n)
                    diff[i][col2+1] -= 1;
            }
        }

        ////Step-2 Now calculate prefix sum
        for(int i = 0; i < n; i++) {
            for(int j = 1; j < n; j++) {
                diff[i][j] += diff[i][j-1];
            }
        }

        return diff;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N \cdot Q + N^2)$ where $Q$ is queries size and $N$ is the grid side length.
- **Space Complexity**: $O(N^2)$ for the grid difference state matrix.

---
**Key Takeaways**:
- **Grid Row Projection**: Applying row-by-row 1D difference arrays is an intuitive, easy-to-implement alternative to full 2D prefix integrations.
