# LC2381. Shifting Letters II

[LeetCode Problem Link](https://leetcode.com/problems/shifting-letters-ii/)

## Intuition & Approach
- We are given a string `s` and range shift queries of the form `[start, end, direction]`, where direction is 0 (backward / -1) or 1 (forward / +1). We want to return the final shifted string.
- Instead of shifting characters one-by-one per query, we use a difference array to accumulate the net shift for each index:
  - For a forward shift, increment `diff[L]` and decrement `diff[R+1]`.
  - For a backward shift, decrement `diff[L]` and increment `diff[R+1]`.
  - Compute the prefix sum of `diff` to get the net shifts at each index.
  - Shift each character in `s` by `(net_shift % 26 + 26) % 26` to handle cycles and negative values.

## Code Implementation
```cpp
class Solution {
public:
    string shiftingLetters(string s, vector<vector<int>>& shifts) {
        int n = s.size();
        vector<int> change(n,0);
        for(auto& shift: shifts){
            int start = shift[0];
            int end = shift[1];
            int direction = shift[2];

            change[start] += (direction == 1)? 1:-1;
            if(end + 1 < n)
            change[end+1] -= (direction == 1)? 1:-1;
        }
        int cumSum = 0;
        for(int i=0;i<n;i++){
            cumSum += change[i];
            int shift = (cumSum % 26);
            if(shift < 0) shift+=26;
            s[i] = ((s[i]-'a') + shift)%26 + 'a';
        }
        return s;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N + Q)$ where $N$ is string length and $Q$ is the shifts size.
- **Space Complexity**: $O(N)$ for the difference array.

---
**Key Takeaways**:
- **Cyclic Net Shifts**: Difference arrays accumulate integer range updates efficiently, which can then be resolved cyclicly using modulus arithmetic.
