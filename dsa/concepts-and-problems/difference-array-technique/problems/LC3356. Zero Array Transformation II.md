# LC3356. Zero Array Transformation II

[LeetCode Problem Link](https://leetcode.com/problems/zero-array-transformation-ii/)

## Intuition & Approach
- This is similar to Zero Array I, but each query is of the form `[l, r, val]` and we want to find the minimum number of queries (prefix of queries) needed to make the array zero.
- Since query effectiveness is monotonic (if $k$ queries are sufficient, any $k' > k$ queries are also sufficient), we can use Binary Search on the Answer (the number of queries $k$):
  - For a candidate $mid$, build a difference array using only the first $mid$ queries.
  - Compute prefix sums and check if the accumulated decrement at each index is $\ge nums[i]$.
  - If yes, search the left half ($r = mid$). Else, search the right half ($l = mid + 1$).

## Code Implementation
```cpp
class Solution {
public:
    bool isZeroArray(vector<int>& nums, vector<vector<int>>& queries,int q){
        int n = nums.size();
        vector<int> changes(n,0);
        for(int i =0;i<q;i++){
            vector<int> query = queries[i];
            int l = query[0];
            int r = query[1];
            int val = query[2];
            changes[l] += -val;
            if(r+1<n) changes[r+1] -= -val;
        }
        for(int i=1;i<n;i++){
            changes[i] = changes[i] + changes[i-1];
        }

        for(int i=0;i<n;i++){
            if(nums[i] + changes[i] > 0) return false;
        }
        return true;
    }
    int minZeroArray(vector<int>& nums, vector<vector<int>>& queries) {
        int q = queries.size();
        int l = 0;
        int r = q;
        int ans = -1;
        while(l<=r){
            int mid = l + (r-l)/2 ;
                
            if(isZeroArray(nums,queries,mid)){
                ans = mid;
                r = mid - 1;
            }
            else{
                l = mid + 1;
            }

        } 
        return ans;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O((N + Q) \log Q)$ using binary search and difference array validation.
- **Space Complexity**: $O(N)$ for the difference array.

---
**Key Takeaways**:
- **Monotonic Query Thresholds**: Combining binary search with prefix-sum validation detects query cutoffs efficiently.
