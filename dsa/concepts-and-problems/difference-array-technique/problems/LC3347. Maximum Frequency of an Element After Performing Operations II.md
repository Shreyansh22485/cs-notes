# LC3347. Maximum Frequency of an Element After Performing Operations II

[LeetCode Problem Link](https://leetcode.com/problems/maximum-frequency-of-an-element-after-performing-operations-ii/)

## Intuition & Approach
- This is similar to Operations I, but the values and array size are much larger, making a map-based prefix sum too slow. We must use sorting and two pointers (sliding window) or binary search.
- We sort `nums`. For any candidate target value $X$ (which is always $nums[i]$, $nums[i]-k$, or $nums[i]+k$):
  - Elements that can change to $X$ must lie in $[X - k, X + k]$.
  - We find the number of elements in this range using two pointers (sliding window) on the sorted array.
  - The frequency is `original_freq[X] + min(numOperations, range_count - original_freq[X])`.
  - We keep track of the maximum frequency over all possible candidates.

## Code Implementation
```cpp
class Solution {
public:
    int maxFrequency(vector<int>& nums, int k, int numOperations) {
        map<int,int> diff;
        unordered_map<int, int> freq;

        for(int i = 0; i < nums.size(); i++) {
            freq[nums[i]]++;
            diff[nums[i]] += 0;
            int l = max(nums[i]-k, 0);
            int r = nums[i]+k;

            diff[l]++;
            diff[r+1]--;
        }


        int result = 1;

        int cumSum = 0;
        for(auto it: diff){
            cumSum += it.second;
            int targetFreq     = freq[it.first];
            int needConversion = cumSum - targetFreq;

            int maxPossibleFreq = min(needConversion, numOperations);

            result = max(result, targetFreq + maxPossibleFreq);           
        }

        return result;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N \log N)$ to sort the array and binary search candidates.
- **Space Complexity**: $O(N)$ to store candidate elements and frequencies.

---
**Key Takeaways**:
- **Sliding Windows on Sorted Coordinates**: Amortized two-pointer sliding window scans can replace difference arrays when coordinate sizes prevent direct indexing.
