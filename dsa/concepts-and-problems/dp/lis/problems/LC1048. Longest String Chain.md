# LC1048. Longest String Chain

[LeetCode Problem Link](https://leetcode.com/problems/longest-string-chain/)

## Intuition & Approach
- Word $A$ is a predecessor of word $B$ if we can insert exactly one letter into $A$ to make it equal to $B$. We want to find the longest chain of words.
- This is a variation of LIS. First, sort all words by length so that predecessor transitions always flow forward.
- We use a hash map `dp` where `dp[word]` stores the longest chain ending at `word`.
- For each word, we check all possible words formed by deleting exactly one character from it. If a predecessor is in our map, we update: `dp[word] = max(dp[word], dp[predecessor] + 1)`. The result is the maximum value in `dp`.

## Code Implementation
```cpp
class Solution {
public:
    static bool comp(const string &a, const string &b) {
        return a.size() < b.size();
    }
    bool compare(string& a ,string& b){
       if(a.size()-b.size()!=1) return false;
       int n=a.size();
       int m=b.size();
       int i=0;
       int j=0;
       while(i<n){
            if(j<m && a[i]==b[j]){
                j++;
            }
            i++;
       }
       if(i==n && j==m) return true;
       return false;
    }
    int longestStrChain(vector<string>& words) {
        int n=words.size();
        vector<int> dp(n,1);
        sort(words.begin(),words.end(),comp);
        int maxi=0;
        
        for(int i=0;i<n;i++){
            for(int prev=0;prev<i;prev++){
                if(compare(words[i],words[prev]) && 1+dp[prev]>dp[i]){
                    dp[i]=1+dp[prev];
                }
            }
            if(dp[i]>maxi){
                maxi=dp[i];
            }
        }
        return maxi;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N \log N + N \cdot L^2)$ where $N$ is the number of words, $L$ is the maximum length of a word (up to 16). Generating predecessors takes $O(L)$ strings of size $L$.
- **Space Complexity**: $O(N \cdot L)$ to store word states in the DP hash map.

---
**Key Takeaways**:
- **Predecessor Deletion Search**: Sorting by string length guarantees correct topological flow. Checking predecessors by deleting a character is much faster than inserting one.
