```cpp
class Solution {
public:
    static bool comp(const string &a, const string &b) {
        return a.size() < b.size();
    }
    bool compare(string& a ,string& b){
       if(a.size()-b.size()!=1) return false;
       int n=a.size();
       int m=b.size();
       int i=0;
       int j=0;
       while(i<n){
            if(j<m && a[i]==b[j]){
                j++;
            }
            i++;
       }
       if(i==n && j==m) return true;
       return false;
    }
    int longestStrChain(vector<string>& words) {
        int n=words.size();
        vector<int> dp(n,1);
        sort(words.begin(),words.end(),comp);
        int maxi=0;
        
        for(int i=0;i<n;i++){
            for(int prev=0;prev<i;prev++){
                if(compare(words[i],words[prev]) && 1+dp[prev]>dp[i]){
                    dp[i]=1+dp[prev];
                }
            }
            if(dp[i]>maxi){
                maxi=dp[i];
            }
        }
        return maxi;
    }
};
```
- Time Complexity: O(n^2 * m) - We are using two nested loops to iterate through the array of words, where n is the number of words. The outer loop runs n times, and for each iteration of the outer loop, the inner loop can run up to n times in the worst case. Additionally, the `compare` function takes O(m) time, where m is the maximum length of a word, leading to an overall time complexity of O(n^2 * m).
- Space Complexity: O(n) - We are using a 1D dp array of size n to store the length of the longest string chain ending at each index, which requires O(n) space. The sorting operation also takes O(n log n) time, but it does not affect the overall space complexity.
