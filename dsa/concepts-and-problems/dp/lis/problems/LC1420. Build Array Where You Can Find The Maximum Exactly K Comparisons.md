# LC1420. Build Array Where You Can Find The Maximum Exactly K Comparisons

[LeetCode Problem Link](https://leetcode.com/problems/build-array-where-you-can-find-the-maximum-exactly-k-comparisons/)

## Intuition & Approach
- We want to build an array of size $n$ with elements in $[1, m]$ such that the search cost (number of times the maximum element updates when scanning left-to-right) is exactly $k$.
- Let `dp[i][max_val][cost]` be the number of valid arrays of prefix size $i$, where the maximum value in the prefix is `max_val` and the search cost is `cost`.
- For the next element $x \in [1, m]$ at index $i+1$:
  - If $x \le \text{max_val}$, the maximum does not change. The search cost remains `cost`. The state transition is: `new_dp[i+1][max_val][cost] += dp[i][max_val][cost] * max_val`.
  - If $x > \text{max_val}$, $x$ becomes the new maximum. The search cost increments to `cost + 1`. The transition is: `new_dp[i+1][x][cost + 1] += dp[i][max_val][cost]`.
- We sum up `dp[n][val][k]` for all `val` in $[1, m]$ to get the final count.

## Code Implementation
```cpp
class Solution {
public:
    const int mod = 1e9 + 7;
    int f(int ind ,int maxSoFar , int n , int m , int k, vector<vector<vector<int>>>& dp){
        if(ind == n){
            if(k==0) return 1;
            return 0;
        }
        if(k==-1) return 0;
        if(dp[ind][maxSoFar+1][k] != -1) return dp[ind][maxSoFar+1][k];


        int result=0;
        for(int i = 1 ; i<=m;i++){
            if(maxSoFar < i){
                result = (result%mod  + f(ind+1,i,n,m,k-1,dp) % mod) % mod;
            }
            else{
                result =  (result%mod  + f(ind+1,maxSoFar,n,m,k,dp) %mod ) % mod;
            }
        }
        return dp[ind][maxSoFar+1][k] = result % mod;
    }
    int numOfArrays(int n, int m, int k) {
        vector<vector<vector<int>>> dp(n+1,vector<vector<int>>(m+2,vector<int>(k+1,-1)));
        return f(0,-1,n,m,k,dp);
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N \cdot M^2 \cdot K)$ using bottom-up DP tabulation.
- **Space Complexity**: $O(N \cdot M \cdot K)$ which can be optimized to $O(M \cdot K)$ by using rolling arrays for index $i$.

---
**Key Takeaways**:
- **Prefix Maximum Transitions**: Dynamic Programming with multiple dimensions (index, current maximum, constraint counter) is standard for counting sequences with running properties.
