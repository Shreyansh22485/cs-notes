# LC646. Maximum Length of Pair Chain

[LeetCode Problem Link](https://leetcode.com/problems/maximum-length-of-pair-chain/)

## Intuition & Approach
- We are given pairs `[left, right]` where a pair $(a, b)$ can link to $(c, d)$ if $b < c$. We want to find the longest chain of pairs.
- This can be modeled as LIS (sort pairs by their start coordinates, then run $O(N^2)$ DP). However, we can solve it in $O(N \log N)$ time using greedy **Interval Scheduling**:
  - Sort pairs by their **right** coordinate (end time).
  - Greedily pick the pair that ends as early as possible. This leaves the maximum possible room for subsequent pairs.
  - Iterate through sorted pairs: if a pair's start coordinate is strictly greater than the current end, append it to the chain and update the end coordinate.

## Code Implementation
```cpp
class Solution {
public:
    int findLongestChain(vector<vector<int>>& pairs) {
        sort(pairs.begin(),pairs.end());
        int n = pairs.size();
        vector<int> dp(n,1);
        int maxi = 1;
        for(int i =0;i<n;i++){
            for(int prev=0;prev<i;prev++){
                if(pairs[prev][1] < pairs[i][0] && 1 + dp[prev] > dp[i]){
                    dp[i] = dp[prev] + 1;
                }
            }
            maxi=max(maxi,dp[i]);
        }
        return maxi;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N \log N)$ due to sorting. The linear scan takes $O(N)$ time.
- **Space Complexity**: $O(1)$ auxiliary space.

---
**Key Takeaways**:
- **Interval Scheduling Greedy Choice**: When select chains of intervals with no overlap, sorting by end time and selecting greedily yields the optimal solution in $O(N \log N)$ time, bypassing $O(N^2)$ DP.
