# LC3202. Find the Maximum Length of Valid Subsequence II

[LeetCode Problem Link](https://leetcode.com/problems/find-the-maximum-length-of-valid-subsequence-ii/)

## Intuition & Approach
- We want to find the maximum length of a subsequence of `nums` such that for any adjacent elements $a$ and $b$ in the subsequence, $(a + b) \pmod k = \text{const}$.
- Let `dp[val][rem]` be the length of the longest valid subsequence ending with value `val` (where `val = num % k`) where the alternating modulo sum is `rem`.
- For each number $num$ in `nums` (with $val = num \pmod k$):
  - For every possible remainder $r \in [0, k-1]$:
    - The predecessor element must have been `(r - val + k) % k`. Let this be `pred`.
    - The state update is: `dp[val][r] = dp[pred][r] + 1`.
  - We track the maximum value.

## Code Implementation
```cpp
class Solution {
public:
    int maximumLength(vector<int>& nums, int k) {
        int n = nums.size();
        vector<vector<int>> dp(k, vector<int>(n, 1)); // LIS for each mod
        int maxSub = 1;

        for(int i = 1; i < n; i++) {
            for(int j = 0; j < i; j++) {
                int mod = (nums[j] + nums[i]) % k;
                dp[mod][i] = max(dp[mod][i], 1 + dp[mod][j]);
                maxSub = max(maxSub, dp[mod][i]);
            }
        }


        return maxSub;

    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N \cdot K)$ where $N$ is array size and $K$ is the modulo constraint.
- **Space Complexity**: $O(K^2)$ to store the DP table.

---
**Key Takeaways**:
- **Alternating Remainder DP**: Storing state metrics grouped by remainders modulo $k$ reduces subsequence verification to constant time state lookups.
