# LC746. Min Cost Climbing Stairs

[LeetCode Problem Link](https://leetcode.com/problems/min-cost-climbing-stairs/)

## Intuition & Approach
- We want to reach the top of a staircase. At each step $i$, we pay `cost[i]` to climb 1 or 2 steps. We can start from index 0 or index 1.
- Let `dp[i]` be the minimum cost to reach step $i$. The recurrence is: `dp[i] = cost[i] + min(dp[i-1], dp[i-2])`.
- We maintain the two preceding costs to determine the next one, updating them iteratively to keep space at $O(1)$.

## Code Implementation
```cpp
class Solution {
public:
    int rob1(vector<int>& nums, int start, int end) {
        int prev2=0;
        int prev1=0;
        int cur;
        for(int i=start;i<=end;i++){
            cur=max(prev1,nums[i]+prev2);
            prev2=prev1;
            prev1=cur;
        }
        return prev1;

        
    }
    int rob(vector<int>& nums) {
        int n=nums.size();
        if(n==1) return nums[0];
        if(n==2) return max(nums[0],nums[1]);

        return max(rob1(nums, 0, n-2), rob1(nums, 1, n-1));
        
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N)$ time.
- **Space Complexity**: $O(1)$ auxiliary space.

---
**Key Takeaways**:
- **Cost Optimization Induction**: Transition states represent the minimal accumulative cost rather than path combinations, requiring a `min` operator instead of a sum.
