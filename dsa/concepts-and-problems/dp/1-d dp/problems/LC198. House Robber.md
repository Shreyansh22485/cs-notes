# LC198. House Robber

[LeetCode Problem Link](https://leetcode.com/problems/house-robber/)

## Intuition & Approach
- We want to rob houses to maximize gold, but robbing adjacent houses triggers alarms.
- For each house $i$, we make a binary choice:
  - Rob house $i$: We accumulate `nums[i] + rob(i - 2)`.
  - Skip house $i$: We keep the value `rob(i - 1)`.
- This yields the recurrence: `dp[i] = max(dp[i-1], nums[i] + dp[i-2])`.
- To optimize space to $O(1)$, we only maintain two variables: `prev2` (representing `dp[i-2]`) and `prev1` (representing `dp[i-1]`). We calculate `curr = max(prev1, nums[i] + prev2)` and shift states.

## Code Implementation
```cpp
class Solution {
public:
    int rob(vector<int>& nums) {
        int n = nums.size();
        //dp[i]=maximum amt of money  in array of length i;
        vector<int> dp(n+1,0);
        dp[0]=0;
        dp[1]=nums[0];

        for(int i=2;i<=n;i++){
            // dp[i]=dp[i-1];
            dp[i]=max(dp[i-1], nums[i-1]+dp[i-2]);
        }
        return dp[n];

        
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N)$ where $N$ is the number of houses.
- **Space Complexity**: $O(1)$ auxiliary space.

---
**Key Takeaways**:
- **Rob or Skip Pattern**: The standard recurrence model for mutually exclusive adjacent selections. Optimize space from $O(N)$ to $O(1)$ by only storing the two preceding DP values.
