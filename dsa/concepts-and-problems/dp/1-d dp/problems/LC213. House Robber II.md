# LC213. House Robber II

[LeetCode Problem Link](https://leetcode.com/problems/house-robber-ii/)

## Intuition & Approach
- This is similar to House Robber I, but the houses are arranged circularly. Robbing the first house means the last house cannot be robbed, and vice versa.
- We can solve this circular constraint by dividing the problem into two linear cases:
  - Case 1: Rob houses from index 0 to $N-2$ (exclude the last house).
  - Case 2: Rob houses from index 1 to $N-1$ (exclude the first house).
- We run the standard House Robber I DP on both slices and return the maximum of the two results.

- **Time Complexity**: $O(N)$ since we run the linear DP twice.

**Key Takeaways**:
- **Circular Array De-circularization**: Circular constraints in array DP can often be resolved by running linear DP on slices (excluding boundaries).
