# LC62. Unique Paths

[LeetCode Problem Link](https://leetcode.com/problems/unique-paths/)

## Intuition & Approach
The goal is to count the number of unique paths from the top-left cell $(0,0)$ to the bottom-right cell $(m-1, n-1)$ in a grid. The robot can only move either down or right at any point.

### Approach 1: Recursion + Memoization
- We define a helper function `f(i, j, dp)` representing the number of paths to reach cell $(i, j)$ from the starting cell $(0,0)$.
- Since the robot can only move down or right, it must reach $(i, j)$ from either the top cell $(i-1, j)$ or the left cell $(i, j-1)$.
- **Recurrence**: `f(i, j) = f(i-1, j) + f(i, j-1)`.
- **Base Cases**:
  - If we reach the starting cell $(0,0)$, there is exactly $1$ way to be there: return 1.
  - If we exceed grid boundaries ($i < 0$ or $j < 0$), there are 0 ways: return 0.
- We cache calculated states in a 2D table `dp` initialized to `-1`.

### Approach 2: Tabulation (Bottom-Up)
- We declare a 2D table `dp` of size $m \times n$ where `dp[i][j]` stores the paths to reach cell $(i, j)$.
- We loop through row index $i$ and column index $j$:
  - If $i == 0$ and $j == 0$, set `dp[0][0] = 1`.
  - Else, `dp[i][j] = dp[i-1][j] (if i > 0) + dp[i][j-1] (if j > 0)`.
- The result is stored in `dp[m-1][n-1]`.

### Approach 3: Combinatorics (Optimal)
- To reach $(m-1, n-1)$ from $(0,0)$, the robot must take exactly $m-1$ down steps and $n-1$ right steps, for a total of $N = m + n - 2$ steps.
- The number of unique paths is the number of ways to arrange these steps, which is given by the combinations formula:
  $$\binom{N}{r} = \binom{m + n - 2}{m - 1} = \frac{(m + n - 2)!}{(m - 1)! \cdot (n - 1)!}$$
- We can compute this directly in $O(\min(m, n))$ time using a simple loop.

## Code Implementation

### Approach 1: Recursion + Memoization
```cpp
class Solution {
public:
    int f(int i, int j, vector<vector<int>>& dp) {
        if (i == 0 && j == 0) return 1;
        if (i < 0 || j < 0) return 0;
        if (dp[i][j] != -1) return dp[i][j];
        
        return dp[i][j] = f(i-1, j, dp) + f(i, j-1, dp);
    }
    int uniquePaths(int m, int n) {
        vector<vector<int>> dp(m, vector<int>(n, -1));
        return f(m-1, n-1, dp);
    }
};
```

### Approach 2: Tabulation (Bottom-Up)
```cpp
class Solution {
public:
    int uniquePaths(int m, int n) {
        vector<vector<int>> dp(m, vector<int>(n, 0));
        
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                if (i == 0 && j == 0) {
                    dp[i][j] = 1;
                } else {
                    int up = 0, left = 0;
                    if (i > 0) up = dp[i-1][j];
                    if (j > 0) left = dp[i][j-1];
                    dp[i][j] = up + left;
                }
            }
        }
        return dp[m-1][n-1];
    }
};
```

### Approach 3: Combinatorics (Optimal)
```cpp
class Solution {
public:
    long long nCr(int n,int r){
        double res=1;
        for(int i=0;i<r;i++){
            res=res*(n-i);
            res=res/(i+1);
        }
        return res;

         int N = n + m - 2;
            int r = m - 1;

            for (int i = 1; i <= r; i++)
                res = res * (N - r + i) / i;
            return (int)res;

};
```

## Complexity Analysis

### Approach 1: Recursion + Memoization
- **Time Complexity**: $O(m \times n)$ to compute the states.
- **Space Complexity**: $O(m \times n)$ for the 2D DP grid + $O(m + n)$ for recursion call stack.

### Approach 2: Tabulation
- **Time Complexity**: $O(m \times n)$ to iterate through all grid cells.
- **Space Complexity**: $O(m \times n)$ to store the DP grid.

### Approach 3: Combinatorics
- **Time Complexity**: $O(\min(m, n))$ to compute the combination count.
- **Space Complexity**: $O(1)$ auxiliary space.

---
**Key Takeaways**:
- **Grid Path Addition**: Unique paths on a grid can be solved recursively by adding path counts from predecessor directions.
- **Combinatorial Simplification**: When steps are strictly monotonic, grid path routing maps directly to combinatorics ($\binom{N}{r}$), yielding a highly optimal $O(1)$ space solution.
