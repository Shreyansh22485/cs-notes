# LC1594. Maximum Non Negative Product in a Matrix

[LeetCode Problem Link](https://leetcode.com/problems/maximum-non-negative-product-in-a-matrix/)

## Intuition & Approach
- The problem asks us to find a path from $(0,0)$ to $(n-1, m-1)$ that maximizes the product of numbers along the path, provided the product is non-negative.
- Since the grid contains negative numbers, a large negative product multiplied by another negative number can yield a large positive product. Similarly, a large positive product multiplied by a negative number can yield a small negative product.
- Thus, at each cell $(i, j)$, we must maintain both the **maximum product** and the **minimum product** to reach that cell.
- **DP State**: We define `dp[i][j] = {maxProduct, minProduct}` as a pair of long long values.
- **State Transitions**:
  - For each cell $(i, j)$, we consider transitions from the left neighbor $(i, j-1)$ and the top neighbor $(i-1, j)$ (if they exist).
  - Let the candidate products be:
    - Left: `grid[i][j] * leftMax` and `grid[i][j] * leftMin`
    - Top: `grid[i][j] * upMax` and `grid[i][j] * upMin`
  - The new maximum product `dp[i][j].first` is the maximum of these candidates, and the new minimum product `dp[i][j].second` is the minimum.
- **Base Case**: At $(0,0)$, `dp[0][0] = {grid[0][0], grid[0][0]}`.
- **Modulo Constraint**: We perform all DP computations using 64-bit integers (`long long`) to prevent overflow, and apply the modulo operation $10^9 + 7$ only at the final step.

## Code Implementation

### Approach 1: Recursion + Memoization
```cpp
class Solution {
public:
    typedef long long ll;
    const int MOD = 1e9+7;
    pair<ll,ll> f(int i, int j, vector<vector<int>>& grid, vector<vector<pair<ll,ll>>>& dp){
        if(i == 0 && j == 0){
            return {grid[i][j],grid[i][j]};
        }

        ll maxProduct = LLONG_MIN;
        ll minProduct = LLONG_MAX;

        if(dp[i][j] != make_pair(LLONG_MIN, LLONG_MAX)) return dp[i][j];
        
        if(j-1>=0){
            auto[leftMaxProduct, leftMinProduct] = f(i,j-1,grid,dp);
            maxProduct = max({maxProduct, grid[i][j]*leftMaxProduct, grid[i][j]*leftMinProduct});
            minProduct = min({minProduct, grid[i][j]*leftMaxProduct, grid[i][j]*leftMinProduct});
        }

        if(i-1>=0){
            auto[upMaxProduct, upMinProduct] = f(i-1,j,grid,dp);
            maxProduct = max({maxProduct, grid[i][j]*upMaxProduct, grid[i][j]*upMinProduct});
            minProduct = min({minProduct, grid[i][j]*upMaxProduct, grid[i][j]*upMinProduct});
        }
        
        return dp[i][j] = {maxProduct, minProduct};
    }

    int maxProductPath(vector<vector<int>>& grid) {
        int n = grid.size();
        int m = grid[0].size();
        vector<vector<pair<ll,ll>>> dp(n,vector<pair<ll,ll>>(m,make_pair(LLONG_MIN, LLONG_MAX)));
        auto [maxProduct, minProduct] = f(n-1,m-1,grid,dp);
        return max(maxProduct,-1ll) % MOD;
    }
};
```

### Approach 2: Tabulation (Bottom-Up)
```cpp
class Solution {
public:
    typedef long long ll;
    const int MOD = 1e9+7;
    int maxProductPath(vector<vector<int>>& grid) {
        int n = grid.size();
        int m = grid[0].size();
        vector<vector<pair<ll,ll>>> dp(n,vector<pair<ll,ll>>(m,make_pair(LLONG_MIN,LLONG_MAX)));

        for(int i=0;i<n;i++){
            for(int j=0;j<m;j++){
                if(i == 0 && j == 0){
                    dp[i][j] = {grid[i][j],grid[i][j]};
                    continue;
                }

                ll maxProduct = LLONG_MIN;
                ll minProduct = LLONG_MAX;
                
                if(j-1>=0){
                    auto[leftMaxProduct, leftMinProduct] = dp[i][j-1];
                    maxProduct = max({maxProduct, grid[i][j]*leftMaxProduct, grid[i][j]*leftMinProduct});
                    minProduct = min({minProduct, grid[i][j]*leftMaxProduct, grid[i][j]*leftMinProduct});
                }

                if(i-1>=0){
                    auto[upMaxProduct, upMinProduct] = dp[i-1][j];
                    maxProduct = max({maxProduct, grid[i][j]*upMaxProduct, grid[i][j]*upMinProduct});
                    minProduct = min({minProduct, grid[i][j]*upMaxProduct, grid[i][j]*upMinProduct});
                }
                
                dp[i][j] = {maxProduct, minProduct};
            }
        }

        auto [maxProduct, minProduct] = dp[n-1][m-1];
        return max(maxProduct,-1ll) % MOD;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(n \times m)$ to populate the DP matrix of size $n \times m$, since each state transition is calculated in $O(1)$ time.
- **Space Complexity**: $O(n \times m)$ to store the 2D DP grid of pairs.

---
**Key Takeaways**:
- **Dual Extremum Tracking**: When products can multiply negative values, maintaining both the maximum and minimum paths is necessary to handle potential sign inversions.
- **Deferred Modulo Operator**: The modulo operation $10^9 + 7$ must only be applied at the final return statement, as premature modulo conversions destroy sign information and magnitude comparisons during DP updates.
