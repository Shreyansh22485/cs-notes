# LC174. Dungeon Game

[LeetCode Problem Link](https://leetcode.com/problems/dungeon-game/)

## Intuition & Approach
The goal is to determine the knight's minimum initial health required to reach the bottom-right corner of the dungeon grid. The knight's health must be strictly positive ($ge 1$) at every step.

- Because the required health at any cell depends on the path forward, solving this with a standard top-left to bottom-right DP is highly complex. Instead, we should solve the problem **backwards** from the destination $(n-1, m-1)$ to the source $(0,0)$.
- Let `dp[i][j]` represent the minimum health required *upon entering* cell $(i, j)$ to reach the destination successfully.
- **Base Case**: At the destination $(n-1, m-1)$:
  - If `dungeon[n-1][m-1] < 0`, we need `-dungeon[n-1][m-1] + 1` health.
  - If it is positive or zero, we only need the baseline of `1` health.
  - Hence: `dp[n-1][m-1] = dungeon[i][j] < 0 ? -dungeon[i][j] + 1 : 1`.
- **State Transitions**:
  - From $(i, j)$, we can move down to $(i+1, j)$ or right to $(i, j+1)$.
  - The health required to move down is `-dungeon[i][j] + dp[i+1][j]`.
  - The health required to move right is `-dungeon[i][j] + dp[i][j+1]`.
  - We take the minimum of these two possibilities: `ans = min(down, right)`.
  - If `ans <= 0`, it means the current room gives us more health than needed, but since our health must always be at least 1, we round the required health up to `1`.
  - **Recurrence**: `dp[i][j] = ans <= 0 ? 1 : ans`.

## Code Implementation

### Approach 1: Recursion + Memoization
```cpp
class Solution {
public:
    int f(int i,int j,vector<vector<int>>& dungeon,vector<vector<int>>& dp){
        int n=dungeon.size();
        int m=dungeon[0].size();
        if(i<0 || j<0 || i>=n  || j>=m) return 1e8;
        if(i==n-1 && j==m-1){
            return dungeon[i][j]<0?-dungeon[i][j]+1:1;
        }
        if(dp[i][j]!=-1) return dp[i][j];
        //down
        int down=-dungeon[i][j]+f(i+1,j,dungeon,dp);

        //right
        int right=-dungeon[i][j]+f(i,j+1,dungeon,dp);

       int ans= min(down,right);
       return dp[i][j]=ans<=0?1:ans;
    }
    
    int calculateMinimumHP(vector<vector<int>>& dungeon) {
        int n=dungeon.size();
        int m=dungeon[0].size();
        vector<vector<int>> dp(n,vector<int>(m,-1));
        return f(0,0,dungeon,dp);
    }
};
```

### Approach 2: Tabulation (Bottom-Up)
```cpp
class Solution {
public:
    int calculateMinimumHP(vector<vector<int>>& dungeon) {
        int n=dungeon.size();
        int m=dungeon[0].size();

        vector<vector<int>> dp(n,vector<int>(m,0));

        for(int i=n-1;i>=0;i--){
            for(int j=m-1;j>=0;j--){
                if(i==n-1 && j==m-1){
                    dp[i][j] = dungeon[i][j]<0?-dungeon[i][j]+1:1;
                    continue;
                }
                int down = 1e9;
                int right = 1e9;
                if(i+1<n)
                    down=-dungeon[i][j]+dp[i+1][j];

                if(j+1<m)
                    right=-dungeon[i][j]+dp[i][j+1];

                int ans= min(down,right);
                dp[i][j]=ans<=0?1:ans;
            }
        }

        return dp[0][0];
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(n \times m)$ to compute all grid state transitions.
- **Space Complexity**: $O(n \times m)$ to store the DP table.

---
**Key Takeaways**:
- **Backward DP Flow**: When optimal decisions depend on future path constraints, reversing the DP direction (iterating from target to source) resolves dependency issues.
- **Clamping Health Bounds**: Enforcing health thresholds at each step (`ans <= 0 ? 1 : ans`) ensures baseline survival limits are met at every intermediate cell.
