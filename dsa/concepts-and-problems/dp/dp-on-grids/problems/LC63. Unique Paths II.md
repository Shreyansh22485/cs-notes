# LC63. Unique Paths II

[LeetCode Problem Link](https://leetcode.com/problems/unique-paths-ii/)

## Intuition & Approach
This is an extension of the Unique Paths problem. Here, the grid contains obstacles, represented by `1`s. The robot cannot step on any cell that has an obstacle.

### Approach 1: Recursion + Memoization
- We define a helper function `f(i, j, obstacleGrid, dp)` representing the number of paths to reach cell $(i, j)$ from the starting cell $(0,0)$.
- If we hit an obstacle (`obstacleGrid[i][j] == 1`) or step out of bounds ($i < 0$ or $j < 0$), there are $0$ valid paths: return 0.
- If we reach the starting cell $(0,0)$, there is exactly $1$ way to start: return 1.
- Otherwise, we sum paths coming from the top direction and the left direction:
  - **Recurrence**: `f(i, j) = f(i-1, j) + f(i, j-1)`.
- We cache states in a 2D `dp` array of size $m \times n$ initialized with `-1`.

### Approach 2: Tabulation (Bottom-Up)
- We declare a 2D table `dp` of size $m \times n$ where `dp[i][j]` stores the paths to reach cell $(i, j)$.
- We loop through row index $i$ and column index $j$:
  - If `obstacleGrid[i][j] == 1`, set `dp[i][j] = 0` (no paths can cross this obstacle).
  - Else if $i == 0$ and $j == 0$, set `dp[0][0] = 1`.
  - Else, `dp[i][j] = dp[i-1][j] (if i > 0) + dp[i][j-1] (if j > 0)`.
- The result is stored in `dp[m-1][n-1]`.

## Code Implementation

### Approach 1: Recursion + Memoization
```cpp
class Solution {
public:
    int f(int i, int j, vector<vector<int>>& obstacleGrid, vector<vector<int>>& dp) {
        if (i < 0 || j < 0 || obstacleGrid[i][j] == 1) return 0;
        if (i == 0 && j == 0) return 1;
        if (dp[i][j] != -1) return dp[i][j];
        
        return dp[i][j] = f(i-1, j, obstacleGrid, dp) + f(i, j-1, obstacleGrid, dp);
    }
    int uniquePathsWithObstacles(vector<vector<int>>& obstacleGrid) {
        int m = obstacleGrid.size();
        int n = obstacleGrid[0].size();
        vector<vector<int>> dp(m, vector<int>(n, -1));
        return f(m-1, n-1, obstacleGrid, dp);
    }
};
```

### Approach 2: Tabulation (Bottom-Up)
```cpp
class Solution {
public:
    int uniquePathsWithObstacles(vector<vector<int>>& obstacleGrid) {
        int m = obstacleGrid.size();
        int n = obstacleGrid[0].size();
        vector<vector<int>> dp(m, vector<int>(n, 0));
        
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                if (obstacleGrid[i][j] == 1) {
                    dp[i][j] = 0;
                } else if (i == 0 && j == 0) {
                    dp[i][j] = 1;
                } else {
                    int up = 0, left = 0;
                    if (i > 0) up = dp[i-1][j];
                    if (j > 0) left = dp[i][j-1];
                    dp[i][j] = up + left;
                }
            }
        }
        return dp[m-1][n-1];
    }
};
```

## Complexity Analysis

### Approach 1: Recursion + Memoization
- **Time Complexity**: $O(m \times n)$ to compute the states.
- **Space Complexity**: $O(m \times n)$ for the 2D DP grid + $O(m + n)$ for recursion call stack.

### Approach 2: Tabulation
- **Time Complexity**: $O(m \times n)$ to iterate through all grid cells.
- **Space Complexity**: $O(m \times n)$ to store the DP grid.

---
**Key Takeaways**:
- **Obstacle Boundary Handling**: Obstacles can be treated as boundaries in recursion or set to 0 in tabulation, seamlessly fitting into the standard grid sum accumulation formula.
