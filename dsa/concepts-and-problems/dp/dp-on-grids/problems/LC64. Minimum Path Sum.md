# LC64. Minimum Path Sum

[LeetCode Problem Link](https://leetcode.com/problems/minimum-path-sum/)

## Intuition & Approach
The goal is to find a path from the top-left cell $(0,0)$ to the bottom-right cell $(n-1, m-1)$ in a grid of non-negative integers that minimizes the sum of all numbers along the path. The robot can only move either down or right.

### Approach 1: Recursion + Memoization
- We define a helper function `f(i, j, grid, dp)` representing the minimum path sum to reach cell $(i, j)$ from $(0,0)$.
- Since the robot can only move down or right, it must reach $(i, j)$ from either the top cell $(i-1, j)$ or the left cell $(i, j-1)$.
- **Recurrence**: `f(i, j) = grid[i][j] + min(f(i-1, j), f(i, j-1))`.
- **Base Cases**:
  - If we reach the starting cell $(0,0)$, there is no more path to trace, so we return `grid[0][0]`.
  - If we exceed grid boundaries ($i < 0$ or $j < 0$), we return a very large value (like `1e8`) so that this invalid path is ignored in the minimization selection.
- We cache states in a 2D `dp` table initialized with `-1`.

### Approach 2: Tabulation (Bottom-Up)
- We declare a 2D table `dp` of size $n \times m$ where `dp[i][j]` stores the minimum path sum to reach cell $(i, j)$.
- We loop through row index $i$ and column index $j$:
  - If $i == 0$ and $j == 0$, set `dp[0][0] = grid[0][0]`.
  - Else:
    - If $i > 0$, `up = grid[i][j] + dp[i-1][j]`; else `up = 1e8`.
    - If $j > 0$, `left = grid[i][j] + dp[i][j-1]`; else `left = 1e8`.
    - `dp[i][j] = min(up, left)`.
- The result is stored in `dp[n-1][m-1]`.

## Code Implementation

### Approach 1: Recursion + Memoization
```cpp
class Solution {
public:
    int f(int i,int j,vector<vector<int>>& grid,vector<vector<int>>& dp){
        int n=grid.size();
        int m=grid[0].size();
        if(i<0 || i>=n || j<0 || j>=m) return 1e8;
        if(i==0 && j==0) return grid[i][j];
        if(dp[i][j]!=-1) return dp[i][j];
        //up
        int up=grid[i][j]+f(i-1,j,grid,dp);
        //left
        int left=grid[i][j]+f(i,j-1,grid,dp);
        return dp[i][j]= min(up,left);
    }
    int minPathSum(vector<vector<int>>& grid) {
        int n=grid.size();
        int m=grid[0].size();
        vector<vector<int>> dp(n,vector<int>(m,-1));
        return f(n-1,m-1,grid,dp);
    }
};
```

### Approach 2: Tabulation (Bottom-Up)
```cpp
class Solution {
public:
    int minPathSum(vector<vector<int>>& grid) {
        int n = grid.size();
        int m = grid[0].size();
        vector<vector<int>> dp(n, vector<int>(m, 0));

        for (int i = 0; i < n; i++) {
            for (int j = 0; j < m; j++) {
                if (i == 0 && j == 0) {
                    dp[i][j] = grid[i][j];
                } else {
                    int up = 1e8;
                    int left = 1e8;
                    if (i > 0) up = grid[i][j] + dp[i-1][j];
                    if (j > 0) left = grid[i][j] + dp[i][j-1];
                    dp[i][j] = min(up, left);
                }
            }
        }
        return dp[n-1][m-1];
    }
};
```

## Complexity Analysis

### Approach 1: Recursion + Memoization
- **Time Complexity**: $O(n \times m)$ to compute all subproblems.
- **Space Complexity**: $O(n \times m)$ for the 2D DP grid + $O(n + m)$ for recursion call stack.

### Approach 2: Tabulation
- **Time Complexity**: $O(n \times m)$ to iterate through all grid cells.
- **Space Complexity**: $O(n \times m)$ to store the DP grid.

---
**Key Takeaways**:
- **Invalid Paths via Infinity**: When minimizing path properties, out-of-bounds transitions are assigned a very large value (like `1e8`) to effectively ignore them in the `min` selection logic.
- **Local Subproblem Dependence**: Grid path optimization is determined purely by local transitions from predecessor neighbors, yielding a clean bottom-up sequence.
