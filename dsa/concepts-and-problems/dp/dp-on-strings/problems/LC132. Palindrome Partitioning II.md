# LC132. Palindrome Partitioning II

[LeetCode Problem Link](https://leetcode.com/problems/palindrome-partitioning-ii/)

## Intuition & Approach
- The problem asks for the minimum cuts needed to partition a string $s$ such that every substring in the partition is a palindrome. If a partition splits a string into $P$ parts, the number of cuts is $P - 1$.
- This is a partition DP problem combined with substring palindrome precomputation:
  - **LPS Lookup Table**: We precompute a 2D boolean grid `t` where `t[i][j]` is `true` if `s[i...j]` is a palindrome. This is filled in $O(N^2)$ time.
  - **1D Partition DP**: Let `dp[i]` be the minimum number of partitions for the suffix `s[i...n-1]`.
  - For each index $i$ from $N-1$ down to 0, we try all possible split points $j \ge i$:
    - If `s[i...j]` is a palindrome (`t[i][j] == true`), then the minimum partitions is `1 + dp[j+1]`.
    - Recurrence: `dp[i] = min_{i <= j < n, t[i][j] == true} (1 + dp[j + 1])`.
  - **Base Case**: `dp[n] = 0` (empty string requires 0 partitions).
- The final minimum cuts is given by `dp[0] - 1`.

## Code Implementation
```cpp
class Solution {
public:
    int n;
    int f(int i, string s, vector<vector<bool>>& t, vector<int>& dp){
        if(i == n){
            return 0;
        }
        if(dp[i]!=-1) return dp[i];
        int minPartitions = INT_MAX;;
        for(int j=i;j<n;j++){
            if(t[i][j]){
                minPartitions = min(minPartitions, 1 + f(j+1,s,t,dp));
            }
        }
        return dp[i] = minPartitions;
    }

    int minCut(string s) {
        n = s.size();

        vector<vector<bool>> t(n,vector<bool>(n,false)); // t[i][j] = true if S[i:j] is palindrome

        for(int L=1;L<=n;L++){
            for(int i=0;i<n-L+1;i++){
                int j = i + L - 1;

                if(L==1){
                    t[i][j] = true;
                }
                else if(L==2 && s[i] == s[j]){
                    t[i][j] = true;
                }
                else{
                    t[i][j] = (s[i]==s[j]) && t[i+1][j-1];
                }
            }
        }
        
        // vector<int> dp(n,-1);
        // return f(0,s,t,dp)-1;

                                
        vector<int> dp(n + 1, 0);// dp[i] = minimum number of palindrome partitions from i to n-1

        // Base case
        dp[n] = 0;

        for (int i = n - 1; i >= 0; i--) {
            int minPart = INT_MAX;

            for (int j = i; j < n; j++) {
                if (t[i][j]) {
                    minPart = min(minPart, 1 + dp[j + 1]);
                }
            }

            dp[i] = minPart;
        }

        return dp[0] - 1;
    }
    
};
```

## Complexity Analysis
- **Time Complexity**: $O(N^2)$ to fill the $N \times N$ palindrome lookup table `t`, and $O(N^2)$ to compute the 1D DP values of size $N$ (since for each state $i$ we iterate $j$ from $i$ to $N$).
- **Space Complexity**: $O(N^2)$ to store the 2D boolean palindrome lookup table `t`, and $O(N)$ to store the 1D DP table.

---
**Key Takeaways**:
- **Partition DP Pattern**: Finding optimal splits of an array can be modeled with a 1D DP state `dp[i]` tracking the optimal suffix solution, combined with inner loops to test all possible next components.
- **Double DP Synergy**: Combining a 2D DP grid (for constraint checks, like palindrome validation) with a 1D DP state (for partition minimization) reduces overall time complexity to $O(N^2)$ by avoiding recursive checks.
