# LC72. Edit Distance

[LeetCode Problem Link](https://leetcode.com/problems/edit-distance/)

## Intuition & Approach
- The problem asks for the minimum number of operations required to convert `word1` to `word2`. The allowed operations are inserting a character, deleting a character, or replacing a character.
- This is a classical string alignment problem solvable using Dynamic Programming:
  - Let `dp[i][j]` be the minimum operations to convert prefix `word1[0...i-1]` to prefix `word2[0...j-1]`.
  - **Base Cases**:
    - If `word1` is empty (length 0), we must insert all characters of `word2`: `dp[0][j] = j`.
    - If `word2` is empty (length 0), we must delete all characters of `word1`: `dp[i][0] = i`.
  - **State Transitions**:
    - If `word1[i-1] == word2[j-1]`, the last characters match, so no operation is needed: `dp[i][j] = dp[i-1][j-1]`.
    - If they differ, we take the minimum of the three operations plus 1:
      - **Delete** the character from `word1`: `dp[i-1][j]`
      - **Insert** the character into `word1`: `dp[i][j-1]`
      - **Replace** the character in `word1`: `dp[i-1][j-1]`
      - Transition: `dp[i][j] = 1 + min({dp[i-1][j], dp[i][j-1], dp[i-1][j-1]})`.
- **Space Optimization**: Since computing the current row `cur` only requires values from the previous row `prev` and the current row's previous elements, we can reduce the space complexity from $O(N \times M)$ to $O(M)$ by using two 1D vectors `prev` and `cur` representing adjacent rows.

```cpp
class Solution {
public:
    int f(int i ,int j,string& word1, string& word2,vector<vector<int>>& dp){
        if(j==0){
            return i;
        }
        if(i==0) return j;
        if(dp[i][j]!=-1) return dp[i][j];

        if(word1[i-1]==word2[j-1]){
            return dp[i][j]=f(i-1,j-1,word1,word2,dp);
        }
        return dp[i][j]=1 + min(f(i-1,j,word1,word2,dp),min(f(i,j-1,word1,word2,dp),f(i-1,j-1,word1,word2,dp)));
    }
    int minDistance(string word1, string word2) {
        int n=word1.size();
        int m=word2.size();
        // vector<vector<int>> dp(n+1,vector<int>(m+1,0));
        // // return f(n,m,word1,word2,dp);
        // for(int i=0;i<=n;i++) dp[i][0]=i;
        // for(int j=0;j<=m;j++) dp[0][j]=j;

        // for(int i=1;i<=n;i++){
        //     for(int j=1;j<=m;j++){
        //         if(word1[i-1]==word2[j-1]){
        //             dp[i][j]=dp[i-1][j-1];
        //         }
        //         else dp[i][j]=1 + min(dp[i-1][j],min(dp[i][j-1],dp[i-1][j-1]));
        //     }
        // }
        // return dp[n][m];

        vector<int> prev(m+1,0);
        vector<int> cur(m+1,0);

        // for(int i=0;i<=n;i++) dp[i][0]=i; first row will have first col 0 , second row will have first col1 ...
        for(int j=0;j<=m;j++) prev[j]=j;

        for(int i=1;i<=n;i++){
            cur[0]=i;
            for(int j=1;j<=m;j++){
                if(word1[i-1]==word2[j-1]){
                    cur[j]=prev[j-1];
                }
                else cur[j]=1 + min(prev[j],min(cur[j-1],prev[j-1]));
            }
            prev=cur;
        }
        return prev[m];

    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N \times M)$ where $N$ and $M$ are lengths of the two strings, representing the nested traversal of the states.
- **Space Complexity**: $O(M)$ auxiliary space using two 1D state tracking vectors of size $M + 1$.

---
**Key Takeaways**:
- **Three-Choice Grid Decisions**: Edit Distance is a classic example of dynamic programming where transitions model three concurrent decisions (insert, delete, replace).
- **1D Space Reduction**: 2D DP grids where transitions only reference `i-1` states can be space-optimized to $O(\min(N, M))$ using rolling rows.
