# LC647. Palindromic Substrings

[LeetCode Problem Link](https://leetcode.com/problems/palindromic-substrings/)

## Intuition & Approach
The goal is to count the total number of palindromic substrings in a given string $s$. We can solve this problem using two different approaches.

### Approach 1: Dynamic Programming (Bottom-Up)
- We define a 2D boolean array `dp[i][j]` which is `true` if the substring `s[i...j]` is a palindrome.
- We iterate over all possible substring lengths $L$ from 1 to $N$:
  - **Length 1** ($i == j$): Any single character is a palindrome, so `dp[i][j] = true`.
  - **Length 2** ($i + 1 == j$): The substring is a palindrome if the two characters are equal: `dp[i][j] = (s[i] == s[j])`.
  - **Length > 2**: The substring is a palindrome if the boundary characters match ($s[i] == s[j]$) and the inner substring is also a palindrome: `dp[i][j] = (s[i] == s[j] && dp[i+1][j-1])`.
- Whenever `dp[i][j]` is `true`, we increment the counter.

### Approach 2: Expand Around Center (Two Pointers)
- A palindrome is symmetric about its center. A string of length $N$ has $2N - 1$ potential centers:
  - $N$ single-character centers (odd-length palindromes, centered at index $i$).
  - $N - 1$ gap centers between adjacent characters (even-length palindromes, centered between index $i$ and $i+1$).
- For each center, we expand outward using two pointers (`left` moving left, `right` moving right) as long as `s[left] == s[right]`.
- Each valid expansion corresponds to one palindromic substring, which we add to our total count. This bypasses the need for $O(N^2)$ state memory.

## Code Implementation

### Approach 1: Dynamic Programming (Bottom-Up)
```cpp
class Solution {
public:
    int countSubstrings(string s) {
        int n = s.size();

        vector<vector<bool>> dp(n,vector<bool>(n,false)); //dp[i][j] is true if Substring S[i:j] is Palindrome
        // 1 length substring = true -> dp[i][i] = true;
        // 2 lenght substring = True if S[i] == S[i+1] so dp[i][i+1] = S[i] == S[i+1]
        // Genric -> dp[i][j] = ((S[i] = S[j]) && dp[i+1][j-1])

        int count = 0;

        for(int L = 1; L <= n; L++){
            for(int i=0; i + L - 1 < n; i++){
                int j = i + L - 1;

                if(i == j){
                    dp[i][j] = true;
                }
                else if(i+1 == j){
                    dp[i][j] = s[i] == s[j];
                }
                else{
                    dp[i][j] = ((s[i] == s[j]) && dp[i+1][j-1]);
                }

                count += dp[i][j];
            }
        }
        return count;
        
    }
};
```

### Approach 2: Expand Around Center (Two Pointers)
```cpp
class Solution {
public:
    int expand(string s, int left, int right){
        int n=s.size();
        int count=0;
        while(left>=0 && right<n && s[left]==s[right]){
            left--;
            right++;
            count++;
        }
        return count;
    }
    
    int countSubstrings(string s) {
        int n=s.size();
        int count=0;

        for(int i=0;i<n;i++){
            int odd=expand(s,i,i);
            int even=expand(s,i,i+1);

            count+=odd+even;
        }
        return count;
    }
};
```

## Complexity Analysis

### Approach 1: Dynamic Programming
- **Time Complexity**: $O(N^2)$ since we iterate over all possible substrings of size $L$ and compute DP transitions in $O(1)$ time.
- **Space Complexity**: $O(N^2)$ to store the 2D boolean DP grid.

### Approach 2: Expand Around Center
- **Time Complexity**: $O(N^2)$ in the worst case (e.g., a string of all identical characters), but typically faster on average.
- **Space Complexity**: $O(1)$ auxiliary space, since no extra arrays or tables are allocated.

---
**Key Takeaways**:
- **Palindrome Center Symmetry**: Palindromes can be expanded outward from their centers. Odd-length palindromes have a single-character center ($i, i$) and even-length palindromes have a two-character center ($i, i+1$).
- **DP vs. Expansion Space Trade-off**: Dynamic programming tracks substring states cleanly in $O(N^2)$ time and space, whereas center expansion solves the problem in identical $O(N^2)$ time but optimized $O(1)$ space by leveraging structural symmetry.
