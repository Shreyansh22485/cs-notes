# LC1312. Minimum Insertion Steps to Make a String Palindrome

[LeetCode Problem Link](https://leetcode.com/problems/minimum-insertion-steps-to-make-a-string-palindrome/)

## Intuition & Approach
The goal is to find the minimum number of insertion steps required to make a given string $s$ a palindrome. We can solve this problem using two different approaches.

### Approach 1: Reduction to Longest Palindromic Subsequence (LPS)
- The largest part of the string that is already a palindrome is its Longest Palindromic Subsequence (LPS).
- The characters that are not part of this subsequence must be matched by inserting corresponding characters.
- Therefore, the minimum number of insertions required is simply $N - \text{LPS}$, where $N$ is the length of the string.
- We find the LPS of $s$ by calculating the LCS of $s$ and its reversed counterpart $s^{rev}$ using space-optimized $DP$ with $O(N)$ auxiliary space.

### Approach 2: Interval DP (Direct Subproblem Tabulation)
- We define `dp[i][j]` as the minimum insertions to make the substring `s[i...j]` a palindrome.
- We iterate over all possible substring lengths $L$ from 1 to $N$:
  - **Length 1** ($i == j$): A single character is already a palindrome, so `dp[i][j] = 0`.
  - **Length > 1**:
    - If the boundary characters match ($s[i] == s[j]$), they do not require new insertions: `dp[i][j] = dp[i+1][j-1]`.
    - If they do not match, we can either match $s[i]$ at the right end (leaving `s[i+1...j]`) or match $s[j]$ at the left end (leaving `s[i...j-1]`). We take the minimum of these two options plus 1: `dp[i][j] = 1 + min(dp[i+1][j], dp[i][j-1])`.
- The final answer is stored in `dp[0][n-1]`.

### Approach 2: Interval DP (Direct Subproblem Tabulation)
```cpp
class Solution {
public:
    int minInsertions(string s) {
        int n = s.size();

        vector<vector<int>> dp(n, vector<int>(n,0)); // dp[i][j] = Minimum Insertion Steps to Make a String S[i:j] Palindrome


        for(int L=1;L<=n;L++){
            for(int i=0;i + L - 1 < n;i++){
                int j = i + L - 1;

                if(i == j){
                    dp[i][j] = 0;
                }
                else if(s[i] == s[j]){
                    dp[i][j] = dp[i+1][j-1];
                }
                else{
                    dp[i][j] = 1 + min(dp[i+1][j], dp[i][j-1]);
                }
            }
        }

        return dp[0][n-1];
    }
};
```

## Complexity Analysis

### Approach 1: LPS Reduction
- **Time Complexity**: $O(N^2)$ where $N$ is the length of the string, representing the nested loops of LCS.
- **Space Complexity**: $O(N)$ auxiliary space for the rolling row arrays.

### Approach 2: Interval DP
- **Time Complexity**: $O(N^2)$ since we iterate over all possible sub-intervals.
- **Space Complexity**: $O(N^2)$ to store the 2D DP grid.

---
**Key Takeaways**:
- **LPS/Palindrome Complementarity**: The minimum insertions to make a string a palindrome is complementary to the length of its LPS ($N - \text{LPS}$), representing matching operations for the unpaired characters.
- **Interval DP Transitions**: Range-based or interval DP computes states from smaller inner intervals (such as $[i+1, j-1]$) to larger outer intervals ($[i, j]$), which requires iterating over the length of the intervals.
