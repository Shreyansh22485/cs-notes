# LC131. Palindrome Partitioning

[LeetCode Problem Link](https://leetcode.com/problems/palindrome-partitioning/)

## Intuition & Approach
The goal is to partition a given string $s$ such that every substring in the partition is a palindrome. We need to find and return all possible valid partitions.

### Approach 1: Backtracking + Simple Palindrome Validation
- We use recursion to generate partitions. Our recursive function `f(ind, s, path, res)` processes splits starting from index `ind`.
- At each step, we try to split the string at every position $j$ from `ind` to $N-1$:
  - If the prefix `s[ind...j]` is a palindrome (checked via a helper function in $O(N)$ time), we append the substring to our active partition list `path` and recursively call `f(j + 1, s, path, res)`.
  - We backtrack by popping the substring from `path`.
- When `ind == s.size()`, we have partitioned the entire string successfully, so we add `path` to our output results `res`.

### Approach 2: Backtracking + DP Table for Palindrome Lookup
- Instead of checking if `s[i...j]` is a palindrome in $O(N)$ time at every recursive step, we can precompute palindrome states for all substrings using a 2D boolean grid `t` of size $N \times N$:
  - Single characters are palindromes: `t[i][i] = true`.
  - Two-character substrings: `t[i][j] = (s[i] == s[j])`.
  - Substrings of length $>2$: `t[i][j] = (s[i] == s[j] && t[i+1][j-1])`.
- In the backtracking phase, we query `t[i][j] == true` in $O(1)$ time to check if a split is valid, which eliminates redundant character comparison traversals.

## Code Implementation

### Approach 1: Backtracking + Simple Palindrome Validation
```cpp
class Solution {
private:
    bool isPalindrome(string s,int start,int end){
        while(start<=end){
            if(s[start++]!=s[end--]) return false;
        }
        return true;
    }
public:
    void f(int ind,string s, vector<string>& path,vector<vector<string>>& res){
        if(ind==s.size()){
            res.push_back(path);
            return ;
        }
        for(int i=ind;i<s.size();i++){
            if(isPalindrome(s,ind,i)){
                path.push_back(s.substr(ind,i-ind+1));
                f(i+1,s,path,res);
                path.pop_back();
            }
        }
    }
    vector<vector<string>> partition(string s) {
        vector<vector<string>> res;
        vector<string> path;
        f(0,s,path,res);
        return res;
    }
};
```

### Approach 2: Backtracking + DP Table for Palindrome Lookup
```cpp
class Solution {
public:

    void solve(string &s, int i, vector<string>& currPartition, vector<vector<bool>>& t, vector<vector<string>>& result) {
        if(i == s.length()) { //I was able to successfully partition the entire string
            result.push_back(currPartition);
            return;
        }

        for(int j = i; j < s.length(); j++) {
            if(t[i][j] == true) { //palindrome
                currPartition.push_back(s.substr(i, j-i+1));
                solve(s, j+1, currPartition, t, result);
                currPartition.pop_back();
            }
        }
    }

    vector<vector<string>> partition(string s) {
        int n = s.length();
        vector<vector<bool>> t(n, vector<bool>(n, false));

        // Initialize the DP table for palindromic substrings
        //t[i][j] = true -> s[i...j] is a palindrome
        
        for (int i = 0; i < n; ++i) {
            t[i][i] = true; //substring of single character is always a palindrome
        }
        
        for (int L = 2; L <= n; ++L) {
            for (int i = 0; i < n - L + 1; ++i) {
                int j = i + L - 1;
                if (s[i] == s[j]) {
                    if (L == 2) {
                        t[i][j] = true;
                    } else {
                        t[i][j] = t[i + 1][j - 1];
                    }
                }
            }
        }

        vector<vector<string>> result;
        vector<string> currPartition;
        solve(s, 0, currPartition, t, result);

        return result;
    }
};
```

## Complexity Analysis

### Approach 1: Basic Backtracking
- **Time Complexity**: $O(N \cdot 2^N)$ in the worst case (e.g., all same characters, since there are $2^{N-1}$ possible ways to partition a string of length $N$, and checking if a substring is a palindrome takes $O(N)$).
- **Space Complexity**: $O(N)$ auxiliary space for the recursion call stack.

### Approach 2: DP-Accelerated Backtracking
- **Time Complexity**: $O(N^2 + N \cdot 2^N)$ as precomputing the DP table takes $O(N^2)$ time, and the backtracking generation still takes $O(N \cdot 2^N)$ in the worst case.
- **Space Complexity**: $O(N^2)$ to store the 2D boolean DP grid.

---
**Key Takeaways**:
- **Backtracking State Separation**: Partitioning a string into variable-length components can be modeled as backtracking where we explore all valid split points.
- **DP-Accelerated Backtracking**: Precomputing substring properties (like palindrome checks) via a DP grid replaces $O(N)$ traversal queries with $O(1)$ table lookups, reducing query overhead during search tree generation.
