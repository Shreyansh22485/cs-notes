# LC516. Longest Palindromic Subsequence

[LeetCode Problem Link](https://leetcode.com/problems/longest-palindromic-subsequence/)

## Intuition & Approach
The goal is to find the length of the longest palindromic subsequence (LPS) in a given string $s$. We can solve this problem using two different approaches.

### Approach 1: Reduction to Longest Common Subsequence (LCS)
- A palindromic subsequence is symmetric, meaning it reads the same forwards and backwards.
- Therefore, the longest palindromic subsequence of $s$ is equivalent to the Longest Common Subsequence (LCS) between $s$ and its reversed counterpart $s^{rev}$.
- We reverse $s$ to get `temp`, then run the standard space-optimized LCS algorithm on `s` and `temp`. This reduces the space complexity to $O(N)$ using two rolling vectors `prev` and `cur`.

### Approach 2: Interval DP (Direct Subproblem Tabulation)
- We define `dp[i][j]` as the length of the LPS in the substring `s[i...j]`.
- We iterate over all possible substring lengths $L$ from 1 to $N$:
  - **Length 1** ($i == j$): Any single character is a palindrome of length 1, so `dp[i][j] = 1`.
  - **Length > 1**:
    - If the boundary characters match ($s[i] == s[j]$), these two characters contribute 2 to the LPS length: `dp[i][j] = 2 + dp[i+1][j-1]` (where `dp[i+1][j-1]` is 0 if $i + 1 > j - 1$).
    - If they do not match, we take the maximum by excluding either $s[i]$ or $s[j]$: `dp[i][j] = max(dp[i+1][j], dp[i][j-1])`.
- The final answer is stored in `dp[0][n-1]`.

### Approach 2: Interval DP (Direct Subproblem Tabulation)
```cpp
class Solution {
public:
    int longestPalindromeSubseq(string s) {
        int n = s.size();

        vector<vector<int>> dp(n, vector<int>(n,0)); // dp[i][j] = LPS of S[i:j]


        for(int L=1;L<=n;L++){
            for(int i=0;i + L - 1 < n;i++){
                int j = i + L - 1;

                if(i == j){
                    dp[i][j] = 1;
                }
                else if(s[i] == s[j]){
                    dp[i][j] = 2 + dp[i+1][j-1];
                }
                else{
                    dp[i][j] = max(dp[i+1][j], dp[i][j-1]);
                }
            }
        }

        return dp[0][n-1];

    }
};
```

## Complexity Analysis

### Approach 1: LCS Reduction
- **Time Complexity**: $O(N^2)$ where $N$ is the length of the string, representing the nested loops of LCS.
- **Space Complexity**: $O(N)$ auxiliary space for the rolling row arrays.

### Approach 2: Interval DP
- **Time Complexity**: $O(N^2)$ since we iterate over all possible sub-intervals.
- **Space Complexity**: $O(N^2)$ to store the 2D DP grid.

---
**Key Takeaways**:
- **Reduction to LCS**: Reversing a string and running LCS against itself is a robust way to find palindromic properties (subsequences) with optimized $O(N)$ space.
- **Interval DP Transitions**: Range-based or interval DP computes states from smaller inner intervals (such as $[i+1, j-1]$) to larger outer intervals ($[i, j]$), which requires iterating over the length of the intervals.
