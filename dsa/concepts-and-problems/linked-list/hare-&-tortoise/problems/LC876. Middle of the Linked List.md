# LC876. Middle of the Linked List

[LeetCode Problem Link](https://leetcode.com/problems/middle-of-the-linked-list/)

## Intuition & Approach
- We want to find the middle node of a linked list. If there are two middle nodes, return the second.
- We use two pointers, `slow` and `fast`, starting at the head.
- `slow` moves one step, and `fast` moves two steps.
- When `fast` reaches the end of the list (`fast == nullptr` or `fast->next == nullptr`), `slow` will be exactly at the middle node of the list.

## Code Implementation
```cpp
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode() : val(0), next(nullptr) {}
 *     ListNode(int x) : val(x), next(nullptr) {}
 *     ListNode(int x, ListNode *next) : val(x), next(next) {}
 * };
 */
class Solution {
public:
    ListNode* middleNode(ListNode* head) {
        if(head==NULL || head->next==NULL) return head;
        ListNode* fast=head;
        ListNode* slow=head;
        while(fast && fast->next){
            fast=fast->next->next;
            slow=slow->next;
        }
        return slow; // slow is the middle node in case of odd number of nodes and the second middle node in case of even number of nodes

        // if we want to return the first middle node in case of even number of nodes, we can use the following code:
        // while(fast && fast->next && fast->next->next){
        //     fast=fast->next->next;
        //     slow=slow->next;
        // }
        // return slow;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N)$ time.
- **Space Complexity**: $O(1)$ auxiliary space.

---
**Key Takeaways**:
- **Fast-Slow Middle Finder**: In a single pass, a $2\times$ runner reaches the end exactly when a $1	imes$ runner reaches the midpoint.
