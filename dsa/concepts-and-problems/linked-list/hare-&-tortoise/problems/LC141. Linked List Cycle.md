# LC141. Linked List Cycle

[LeetCode Problem Link](https://leetcode.com/problems/linked-list-cycle/)

## Intuition & Approach
- We want to check if a linked list contains a cycle.
- We use Floyd's Cycle-Finding Algorithm (Hare & Tortoise / two pointers):
  - We declare two pointers, `slow` and `fast`, both starting at the head of the list.
  - `slow` moves one step at a time (`slow = slow->next`).
  - `fast` moves two steps at a time (`fast = fast->next->next`).
  - If there is a cycle, the `fast` pointer will eventually enter the cycle and catch up to the `slow` pointer, meeting at the exact same node (`slow == fast`).
  - If `fast` reaches the end of the list (`nullptr`), there is no cycle, so we return `false`.

## Code Implementation
```cpp
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode(int x) : val(x), next(NULL) {}
 * };
 */
class Solution {
public:
    bool hasCycle(ListNode *head) {
        if(head==NULL || head->next==NULL) return false;
        ListNode* slow=head;
        ListNode* fast=head;
        
        while(fast!=NULL && fast->next!=NULL){
            slow=slow->next;
            fast=fast->next->next;
            if(slow==fast) return true;
        }
        return false;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N)$ where $N$ is the number of nodes in the list.
- **Space Complexity**: $O(1)$ auxiliary space.

---
**Key Takeaways**:
- **Floyd's Fast-Slow Pointer**: In circular structures, two runners moving at different speeds are guaranteed to meet if a cycle exists.
