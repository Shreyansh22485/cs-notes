# LC287. Find the Duplicate Number

[LeetCode Problem Link](https://leetcode.com/problems/find-the-duplicate-number/)

## Intuition & Approach
- We are given an array of size $N+1$ containing integers in $[1, N]$. Exactly one number is duplicated. We must find the duplicate without modifying the array and using $O(1)$ space.
- We can model the array as a directed graph where each index $i$ has an edge to $nums[i]$.
- Since the values are in $[1, N]$, index 0 will point to some node, and because of the duplicate, there will be a cycle. The starting node of the cycle represents the duplicated number.
- We use Floyd's cycle detection:
  - `slow = nums[0]`, `fast = nums[nums[0]]`.
  - Loop until `slow == fast`.
  - Reset `slow` to `0` and move both pointers one step at a time (`slow = nums[slow]`, `fast = nums[fast]`) until they meet. The meeting node is the duplicate.

## Code Implementation
```cpp
class Solution {
public:
    int findDuplicate(vector<int>& nums) {
        int n=nums.size();
        int slow=nums[0];
        int fast=nums[0];
        do{
            slow=nums[slow];
            fast=nums[nums[fast]];
        }while(slow!=fast);
        fast=nums[0];
        while(slow!=fast){
            slow=nums[slow];
            fast=nums[fast];
        }
        return slow;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(N)$ time.
- **Space Complexity**: $O(1)$ auxiliary space.

---
**Key Takeaways**:
- **Array as Graph Pointers**: When array values fall within array index boundaries, we can treat $i \to nums[i]$ as pointer transitions, converting duplicate checks into cycle entry detection.
