# LC202. Happy Number

[LeetCode Problem Link](https://leetcode.com/problems/happy-number/)

## Intuition & Approach
- A happy number is a number that eventually reaches 1 when replaced by the sum of squares of its digits. If it loops endlessly in a cycle, it is not happy.
- This is equivalent to cycle detection in a state transition graph where each state $x$ maps to $f(x)$ (sum of squares of digits).
- We can use Floyd's cycle detection (fast-slow pointers):
  - `slow` calculates the sum of squares of digits once: `slow = getNext(slow)`.
  - `fast` calculates it twice: `fast = getNext(getNext(fast))`.
  - If they meet at a value other than 1, a cycle exists, and the number is not happy. If they meet at 1, the number is happy.

## Code Implementation
```cpp
class Solution {
public:
    int sumOfSquare(int n){
        int sum=0;
        while(n!=0){
            int d=n%10;
            sum+=d*d;
            n=n/10;
        }
        return sum;

    }
    bool isHappy(int n) {
        int slow=n;
        int fast=n;
        do{
            slow=sumOfSquare(slow);
            fast=sumOfSquare(sumOfSquare(fast));

        }while(slow!=fast);
        return slow==1;
    }
};
```

## Complexity Analysis
- **Time Complexity**: $O(\log N)$ as the number of digits decreases logarithmically.
- **Space Complexity**: $O(1)$ auxiliary space.

---
**Key Takeaways**:
- **Implicit Cycle Detection**: Floyd's pointer algorithm applies to any state machine or functional transition graph to detect cycles using $O(1)$ space.
