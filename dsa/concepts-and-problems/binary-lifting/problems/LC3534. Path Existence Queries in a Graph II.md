# LC3534. Path Existence Queries in a Graph II

[LeetCode Problem Link](https://leetcode.com/problems/path-existence-queries-in-a-graph-ii/)

## Intuition & Approach
- The problem asks us to find the minimum number of steps (hops) to reach node $v$ from node $u$, or determine if it is unreachable (-1). A hop from node $A$ to node $B$ is valid if their value difference is within `maxDiff`.
- To minimize the total number of hops, we should greedily jump to the farthest reachable node in each step.
- Since we want to query ranges based on values, we sort the array and map node indices:
  - We create `numsIdx` containing `{nums[i], i}` and sort it in ascending order.
  - We map original indices to sorted indices in the `toIdx` array.
- For each sorted node index, we find the farthest reachable node index in a single hop within the value range `value + maxDiff` using binary search (`findFarthestNodeOneHop`). This forms our base transition: `ancestorTable[node][0] = farthestNodeOneHop`.
- We precompute the binary lifting table `ancestorTable` of size $N \times (\log_2 N + 1)$, where `ancestorTable[node][j]` represents the node reached by taking $2^j$ optimal hops from `node`.
  - **Recurrence**: `ancestorTable[node][j] = ancestorTable[ancestorTable[node][j-1]][j-1]`.
- For each query $(u, v)$ (mapped to sorted positions $a, b$, assuming $a < b$):
  - We jump using binary lifting as long as the target reached is strictly less than $b$. We accumulate the jump count.
  - At the end of the lifts, if one additional hop from `curr` can reach or exceed $b$ (`ancestorTable[curr][0] >= b`), we return `jumps + 1`; otherwise, $b$ is unreachable, so we return `-1`.

## Code Implementation
```cpp
class Solution {
public:
    int findFarthestNodeOneHop(vector<pair<int,int>>& numsIdx, int x){ 
        int n = numsIdx.size();
        int l = 0;
        int r = n-1;
        int ans = 0;

        while(l<=r){
            int mid =  (l+r)/2;

            if(numsIdx[mid].first <= x){
                ans = mid;
                l = mid + 1;
            }
            else{
                r = mid - 1;
            }
        }
        return ans;
    }
    vector<int> pathExistenceQueries(int n, vector<int>& nums, int maxDiff, vector<vector<int>>& queries) {
        vector<pair<int,int>> numsIdx;
        vector<int> toIdx(n);

        for(int i=0;i<n;i++){
            numsIdx.push_back({nums[i],i});
        }

        sort(numsIdx.begin(),numsIdx.end());

        for(int i=0;i<n;i++){
            toIdx[numsIdx[i].second] = i;
        }

        int rows = n;
        int cols = log2(n) + 1;
        vector<vector<int>> ancestorTable(rows,vector<int>(cols,0)); // ancestorTable[node][j] -> node that is reached by hopping 2^j

        //filling the j=0 column
        for(int node=0;node<rows;node++){
            int num = numsIdx[node].first;
            int farthestNodeOneHop = findFarthestNodeOneHop(numsIdx, num+maxDiff);
            ancestorTable[node][0] = farthestNodeOneHop;
        }

        for(int j=1;j<cols;j++){
            for(int node=0;node<rows;node++){
                ancestorTable[node][j] = ancestorTable[ ancestorTable[node][j-1] ][j-1];
            }
        }

        vector<int> res;
        for(auto& query: queries){
            int u = query[0];
            int v = query[1];

            int a = toIdx[u];
            int b = toIdx[v];

            if(a == b) {
                res.push_back(0);
                continue;
            }

            if(a > b){
                swap(a,b);
            }
            int curr  = a;
            int jumps = 0;

            for(int j = cols-1; j >= 0; j--) { 
                if(ancestorTable[curr][j] < b) {
                    curr = ancestorTable[curr][j];
                    jumps += (1 << j); //pow(2, j)
                }
            }

            if(ancestorTable[curr][0] >= b){
                res.push_back(jumps+1);
            }
            else{
                res.push_back(-1);
            }
        }
        return res; 
    }
};
```

## Complexity Analysis
- **Time Complexity**:
  - Preprocessing: $O(N \log N)$ to sort and build the binary lifting table.
  - Queries: $O(Q \log N)$ as each query lifts the node in logarithmic steps.
- **Space Complexity**: $O(N \log N)$ auxiliary space to store the binary lifting table.

---
**Key Takeaways**:
- **Greedy Path Optimization**: Shortest paths in monotonic systems (like value-bounded ranges) are found by greedily hopping to the farthest possible state.
- **Binary Lifting on Functional Graphs**: Binary lifting is a generalized jump technique that applies to any functional transition graph $x \to f(x)$ to query arbitrary-sized leaps in logarithmic time.
